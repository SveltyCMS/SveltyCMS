# Radio Widget

<blockquote>**Single-choice selection widget** with radio buttons for selecting one option from a predefined list with strict validation.</blockquote>

## Overview

The **Radio widget** provides an intuitive interface for single-choice selection using radio buttons. It's ideal when you need users to select exactly one option from a small, visible list (typically 2-7 options). Radio buttons make all options immediately visible, unlike dropdowns.

### Key Features

- ‚úÖ **Single Selection**: Only one option can be selected at a time
- ‚úÖ **Strict Validation**: Valibot ensures selected value matches allowed options
- ‚úÖ **Visual Options**: All choices visible without interaction
- ‚úÖ **Accessible**: Native fieldset/legend markup with ARIA support
- ‚úÖ **Dynamic Options**: Configure label/value pairs
- ‚úÖ **Clean Storage**: Stores primitive value (string, number, boolean)
- ‚úÖ **Non-Translatable**: Selection values are universal
- ‚úÖ **Required Support**: Can enforce selection

---

## When to Use Radio Widget

### ‚úÖ Use Radio When:

- **Few Options**: 2-7 choices that fit on screen
- **Single Choice**: User must pick exactly one option
- **Visible Choices**: All options should be immediately visible
- **Binary Decisions**: Yes/No, True/False, Male/Female
- **Status Selection**: Draft/Published, Active/Inactive
- **Priority Levels**: Low/Medium/High
- **User Preferences**: Theme, Language, Notification settings

### ‚ùå Don't Use Radio When:

- **Many Options**: Use `Select` dropdown for 8+ options
- **Multiple Choices**: Use `Checkbox` for multi-select
- **On/Off Toggle**: Use `Toggle` or `Checkbox` for single boolean
- **Search Needed**: Use `Select` with search for large lists
- **Dynamic Options**: Use `Relation` for database-driven options

## Basic Usage

### Simple Status Selection

```typescript
import Radio from '@src/widgets/core/radio';

export const postsCollection = {
	name: 'Posts',
	fields: [
		Radio({
			label: 'Status',
			db_fieldName: 'status',
			required: true,
			options: [
				{ label: 'Draft', value: 'draft' },
				{ label: 'Published', value: 'published' },
				{ label: 'Archived', value: 'archived' }
			]
		})
	]
};
```

## Configuration Properties

| Property       | Type       | Required  | Default     | Description                         |
| -------------- | ---------- | --------- | ----------- | ----------------------------------- |
| `label`        | `string`   | ‚úÖ Yes    | -           | Display label for the field         |
| `db_fieldName` | `string`   | ‚úÖ Yes    | -           | Database field name                 |
| `options`      | `Option[]` | ‚úÖ Yes    | `[]`        | Array of label/value pairs          |
| `required`     | `boolean`  | ‚ùå No     | `false`     | Whether selection is mandatory      |
| `helper`       | `string`   | ‚ùå No     | `undefined` | Helper text                         |
| `translated`   | `boolean`  | üîí Locked | `false`     | Always false (values are universal) |

### Option Interface

```typescript
interface Option {
	label: string; // Display text shown to user
	value: any; // Primitive value stored in database (string, number, boolean)
}
```

## Data Structure

### Storage Format

Radio stores the selected value as a primitive:

```typescript
// MongoDB Document
{
  "_id": "post_123",
  "title": "My Post",
  "status": "published",     // ‚úÖ String value
  "priority": "high",        // ‚úÖ String value
  "featured": true,          // ‚úÖ Boolean value
  "categoryId": 42,          // ‚úÖ Number value
  "createdAt": ISODate("2025-01-15T10:00:00Z")
}
```

### Value Types

```typescript
// String values
options: [
	{ label: 'Draft', value: 'draft' },
	{ label: 'Published', value: 'published' }
];
// Stores: "draft" or "published"

// Number values
options: [
	{ label: 'Low', value: 1 },
	{ label: 'High', value: 5 }
];
// Stores: 1 or 5

// Boolean values
options: [
	{ label: 'Yes', value: true },
	{ label: 'No', value: false }
];
// Stores: true or false
```

## Validation

Radio widget uses Valibot for strict validation:

```typescript
// Automatic validation ensures value matches one of the allowed options
Radio({
	options: [
		{ label: 'A', value: 'a' },
		{ label: 'B', value: 'b' }
	]
});

// Valid: 'a' or 'b'
// Invalid: 'c', null (if required), undefined (if required)

// Error message: "Please select a valid option"
```

## Best Practices

### ‚úÖ Do's

1. **Keep Options Short**

   ```typescript
   options: [
   	{ label: 'Yes', value: 'yes' },
   	{ label: 'No', value: 'no' }
   ];
   ```

2. **Use for 2-7 Options**

   ```typescript
   // Perfect for radio
   options: [
   	{ label: 'Small', value: 's' },
   	{ label: 'Medium', value: 'm' },
   	{ label: 'Large', value: 'l' }
   ];
   ```

3. **Provide Clear Labels**
   ```typescript
   options: [
   	{ label: 'Active (currently in use)', value: 'active' },
   	{ label: 'Inactive (not in use)', value: 'inactive' }
   ];
   ```

### ‚ùå Don'ts

1. **Don't Use for Many Options**

   ```typescript
   // ‚ùå Too many options for radio
   options: Array.from({ length: 50 }, (_, i) => ({
     label: `Option ${i}`,
     value: i
   }))

   // ‚úÖ Use Select instead
   Select({ options: [...] })
   ```

2. **Don't Use for Multi-Select**

   ```typescript
   // ‚ùå Radio can only select one

   // ‚úÖ Use Checkboxes for multiple
   CheckboxGroup({ options: [...] })
   ```

## Radio vs Select vs Toggle

| Feature        | Radio                | Select               | Toggle/Checkbox |
| -------------- | -------------------- | -------------------- | --------------- |
| **Options**    | 2-7 visible          | 8+ in dropdown       | Single boolean  |
| **Selection**  | Single choice        | Single choice        | On/Off          |
| **Visibility** | All visible          | Hidden until click   | Single toggle   |
| **Space**      | Takes vertical space | Compact              | Very compact    |
| **Scanning**   | Easy to scan         | Requires interaction | Immediate       |

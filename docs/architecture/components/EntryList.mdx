# EntryList Component

The `EntryList` is the primary component for displaying collection data in a tabular format with enterprise features like server-side search, filtering, and intelligent preloading.

---

## Architecture

```mermaid
flowchart LR
    subgraph Server["Server (+page.server.ts)"]
        DB[(Database)]
        SC[Server Cache]
    end

    subgraph EntryList["EntryList Component"]
        direction TB
        TH[Table Header]
        TR[Table Rows]
        FI[Filters]
        PG[Pagination]
        MB[MultiButton]
    end

    subgraph Preload["Hover Preloading"]
        HP[preloadData]
        PC[Preload Cache]
    end

    DB --> SC
    SC --> EntryList
    TR -->|Hover| HP
    HP --> PC
```

---

## Features

| Feature                | Description                                   |
| ---------------------- | --------------------------------------------- |
| **Server-Side Search** | Real-time search with debounced queries       |
| **Field Filtering**    | Per-column filtering options                  |
| **Smart Pagination**   | Configurable page sizes (10, 25, 50, 100)     |
| **Hover Preloading**   | Connection-aware speculative loading          |
| **Bulk Selection**     | Multi-item selection with Shift+Click         |
| **Sortable Columns**   | Click-to-sort with ASC/DESC toggle            |
| **Status Display**     | Visual status badges (Published, Draft, etc.) |

## Data Flow

```mermaid
sequenceDiagram
    participant U as User
    participant EL as EntryList
    participant URL as URL State
    participant S as Server

    U->>EL: Change filter/search/page
    EL->>URL: Update URL params
    URL->>S: Request new data
    S-->>EL: Return entries + pagination
    EL->>U: Render updated table

    Note over EL,U: Hover Preloading
    U->>EL: Hover over row
    EL->>S: preloadData(entry URL)
    S-->>EL: Cache response
```

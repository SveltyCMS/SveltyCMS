# API Security & Token Hardening

SveltyCMS employs a zero-trust architecture for API access, combining high-entropy cryptographic tokens with strict multi-tenant isolation.

## 1. Token Architecture

### Website Tokens (API Keys)

Designed for machine-to-machine communication, Website Tokens are stateless secrets bound to a specific `tenantId` and role.

- **Types**:
  - `content-api`: Read-only or restricted access to collection data.
  - `admin-api`: Full administrative access to the system.
- **Storage**: Hashed via SHA-256 before persistence.
- **Optimization**: Uses a single-pass `getByToken` adapter method to avoid redundant database round-trips.

## 2. Authentication Flow

The authentication middleware (`handle-authentication.ts`) prioritizes performance without compromising security.

### Bearer Token Validation

1. **Extraction**: Retrieve token from `Authorization: Bearer <token>` header.
2. **Lookup**: Single database query using `dbAdapter.system.websiteTokens.getByToken`.
3. **Normalization**: Automatically handles legacy tokens by defaulting to `content-api`.
4. **Virtual Identity**: Instead of hydrating a full User document, the system creates a high-performance **Virtual User** context:
   ```typescript
   locals.user = {
     _id: `token:${token._id}`,
     role: token.type === "admin-api" ? "admin" : "guest",
     tenantId: token.tenantId ?? locals.tenantId,
     isApiToken: true,
   };
   ```

## 3. Multi-Tenant Hardening

### Orphaned Token Protection

If a token's owner is deleted (`tenantId: null`), the system automatically isolates the token to the current request's tenant context, preventing cross-tenant privilege escalation.

### Isolation Checks

Every request validated via a Bearer token is strictly verified against the `tenantId` of the current environment. Mismatched tokens are rejected immediately.

## 4. Performance Benchmarks

The hardened token engine achieves:

- **Sub-1ms** validation latency for local adapters.
- **Zero-roundtrip** lookups when using a cache-hydrated `LocalCMS`.
- **Atomic consumption** for one-time tokens to prevent TOCTOU race conditions.

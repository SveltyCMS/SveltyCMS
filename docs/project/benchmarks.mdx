# Performance Benchmarks (April 2026 Update)

> [!CAUTION]
> **Data Safety & Isolation**: All benchmarks are conducted within a strictly isolated **Test Environment**. SveltyCMS utilizes dedicated test databases (`SveltyCMS_test`) and separate configuration files (`private.test.ts`). **Actual live user data is never accessed, modified, or overwritten during any performance test.**

> [!NOTE]
> **April 2026 Performance Overhaul**: Benchmarks have been updated for the Intel-based "Harmonized" architecture. We now utilize professional statistical distributions (p95, p99) to ensure enterprise-grade consistency.

With the introduction of the **Harmonized API Architecture**, SveltyCMS has transitioned to zero-latency direct memory calls.

## 🔬 High-Resolution Methodology

To provide "Enterprise-Ready" transparency, our April 2026 suite moves beyond simple averages.

### 1. Percentiles (p50, p95, p99)
*   **Why**: Averages hide "jitter" (outliers). 99 fast requests and 1 slow request can result in a misleading average.
*   **Insight**: **p95** represents the "worst-case" experience for 95% of users. **p99** identifies critical bottlenecks or Garbage Collection (GC) pauses that impact high-traffic stability.

### 2. Throughput (RPS - Requests Per Second)
*   **Why**: Latency measures speed; RPS measures **capacity**.
*   **Insight**: High RPS demonstrates that SveltyCMS can handle massive traffic spikes without crashing, vital for global edge deployments.

## 🏁 Analysis: Memory Speed vs. Network Speed

The **Harmonized API** effectively eliminates the "Hydration Tax" and "Serialization Tax" for internal CMS operations.

1.  **Zero Serialization**: Data passed to the Local SDK remains as typed TypeScript objects, avoiding `JSON.stringify` and `JSON.parse` cycles.
2.  **No Handshakes**: By bypassing HTTP, we skip TCP/TLS handshakes and routing lookups for every internal request.
3.  **Unified Caching**: The SDK shares the same cache instance as the UI, ensuring 100% cache hits for recently modified data.

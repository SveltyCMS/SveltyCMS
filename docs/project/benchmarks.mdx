# Performance Benchmarks (March 2026)

> [!NOTE]
> **April 2026 Competitive Benchmark Update**: A comprehensive comparison vs Payload 3.81, Directus 11.17, and Strapi 5.41 is currently in progress. Target: 3,000+ aggregate RPS across all adapters.

With the introduction of the **Harmonized API Architecture** and the **LocalCMS SDK**, SveltyCMS has transitioned from traditional HTTP-based internal communication to zero-latency direct memory calls.

> [!IMPORTANT]
> **Internal vs. External Performance**: The dramatic **46x - 300x gains** documented below apply specifically to **internal calls** made via the LocalCMS SDK (same-process). These speedups are achieved by completely bypassing the HTTP round-trip, avoiding JSON serialization/deserialization, and using direct TypeScript function calls. External clients hitting the API via the HTTP Gateway will still see significant improvements (2-5x), but are limited by network latency and transport overhead.

## 🏁 Analysis: Memory Speed vs. Network Speed

The **Harmonized API** effectively eliminates the "Hydration Tax" and "Serialization Tax" for internal CMS operations.

1.  **Zero Serialization**: Data passed to the Local SDK remains as typed TypeScript objects, avoiding `JSON.stringify` and `JSON.parse` cycles.
2.  **No Handshakes**: By bypassing HTTP, we skip TCP/TLS handshakes and routing lookups for every internal request.
3.  **Unified Caching**: The SDK shares the same cache instance as the UI, ensuring 100% cache hits for recently modified data.

# Administration Theme System Architecture (Gin-Inspired)

This document is the strategic blueprint for making SveltyCMS's admin UI as flexible, opinionated, and extensible as **Drupal Gin** — and then going further, thanks to Svelte 5's reactive component model.

---

## 1. Vision: Gin-Level Flexibility

Drupal Gin has become the gold standard for modern admin UIs because it goes beyond color theming — it delivers a cohesive, modular, and deeply customizable experience with dedicated modules for every part of the admin interface.

SveltyCMS aims to match or exceed each:

| Gin Module             | SveltyCMS Equivalent                                 | Priority |
| :--------------------- | :--------------------------------------------------- | :------- |
| **Gin Toolbar**        | Enhanced top + side navigation with role-based menus | High     |
| **Gin Login**          | Branded, fully themed login + forgot password pages  | High     |
| **Gin Layout Builder** | Themed + improved Layout/Block system                | Medium   |
| **Gin Type Tray**      | Content type browser / quick-create tray             | High     |
| **Gin Dashboard**      | Widget-based, customizable home dashboard            | High     |

We achieve this through **color theming** (Skeleton.dev compatible) + **structural theming** (CSS variables + Svelte 5 Context) — with zero PHP/Twig overhead.

### SveltyCMS Advantages Over Gin

- **True reactive Svelte components** → much smoother UX than Drupal's PHP + Twig render pipeline
- **Easier per-tenant / multi-site theming** via native `tenantId` isolation
- **Better mobile responsiveness** out of the box (no Drupal legacy baggage)
- **Live preview** in the theme customizer (possible natively with Svelte 5 `$state`)
- **Zero compilation overhead** — CSS variables swap instantly at runtime

````mermaid
## 3. Pillar A: Design Tokens — `--admin-*` Structural Variables

Structural token defaults are defined as CSS custom properties in `src/utilities.css` (imported by `app.css`), providing static fallback values for the `:root` and `[data-density]` levels. The reactive runtime source of truth is the `AdminTheme` class in `theme-context.svelte.ts`, which exposes these tokens as JavaScript getters consumed by native Svelte 5 components.

```
/* src/utilities.css — :root defaults (cozy density) */
:root {
  --admin-density: 1;

  /* Radii */
  --admin-radius-base: 0.75rem;
  --admin-radius-card: 0.75rem;
  --admin-radius-input: 0.375rem;
  --admin-radius-button: 0.25rem;

  /* Borders & Shadows */
  --admin-border-width: 1px;
  --admin-shadow-elevation: 0 1px 3px 0 rgb(0 0 0 / 0.1);
  --admin-shadow-elevated: 0 10px 15px -3px rgb(0 0 0 / 0.05);

  /* Layout Sizing */
  --admin-sidebar-width: 240px;
  --admin-header-height: 64px;

  /* Features */
  --admin-sticky-bar-height: 56px;

  /* Editorial Status Colors */
  --admin-color-draft: oklch(65% 0 219deg);
  --admin-color-published: oklch(69.59% 0.15 162.47deg);
  --admin-color-scheduled: oklch(58.74% 0.2 256.12deg);
  --admin-color-review: oklch(76.86% 0.16 70.07deg);
}

[data-density="compact"] {
  --admin-density: 0.85;
  --admin-radius-base: 0.25rem;
  --admin-radius-card: 6px;
  --admin-radius-input: 4px;
  --admin-radius-button: 0.125rem;
  --admin-sidebar-width: 200px;
  --admin-header-height: 48px;
  --admin-sticky-bar-height: 44px;
}

[data-density="spacious"] {
  --admin-density: 1.25;
  --admin-radius-base: 1rem;
  --admin-radius-card: 16px;
  --admin-radius-input: 10px;
  --admin-radius-button: 0.625rem;
  --admin-sidebar-width: 300px;
  --admin-header-height: 72px;
  --admin-sticky-bar-height: 64px;
}
```text

Components read these directly with fallbacks:

```
<div class="card shadow-sm"
     style="border-radius: var(--admin-radius-card, 0.75rem);
            border-width: var(--admin-border-width, 1px);">
```text

## 4. Pillar B: Svelte 5 `AdminTheme` Context

The `AdminTheme` class in `src/components/ui/theme-context.svelte.ts` is the reactive source of truth for structural theme state — consumed by all 37+ native UI components via Svelte context.

```
// src/components/ui/theme-context.svelte.ts
export interface ThemeFeatures {
  stickyActionBar: boolean;
  collapsibleSidebar: boolean;
  brandedLogin: boolean;
  highContrastMode: boolean;
  reducedMotion: boolean;
  layoutRegions?: {
    collections: "left" | "right" | "both" | "floating";
    mediaGalleries: "left" | "right" | "both" | "hidden";
  };
}

export interface LayoutState {
  leftSidebar: "full" | "hidden";
  rightSidebar: "full" | "hidden";
  pageheader: "full" | "hidden";
  pagefooter: "full" | "hidden";
  header: "full" | "hidden";
  footer: "full" | "hidden";
}

export interface ThemeConfig {
  id: string;
  name: string;
  density: "compact" | "cozy" | "spacious";
  variant: "flat" | "bordered" | "elevated";
  accentMode: "default" | "primary-only" | "custom";
  role: "editor" | "reviewer" | "translator" | "admin" | "manager";
  themeName: string;
  customCss?: string;
  features: ThemeFeatures;
}
```text

Set in `(app)/+layout.svelte`, consumed in every native component:

```
<!-- src/routes/(app)/+layout.svelte -->
<script lang="ts">
  import { setThemeContext } from '@components/ui/theme-context.svelte';

  const theme = setThemeContext({ density: 'cozy', variant: 'bordered' });
</script>

<div data-density={theme.density} data-admin-theme={theme.themeName}>
  {@render children()}
</div>
```text

**Fallback Guard**: If `getThemeContext()` returns `undefined` (unit tests, setup screens), components resolve to the standard `cozy/bordered` defaults automatically.

## 6. Pillar C: Theme File Format (`/themes/*.json`)

Each theme can be distributed as a JSON file in the `/themes/` directory. Vite's `sveltyCmsPlugin` watches these files and auto-imports them to the database on change.

```
{
  "name": "Developer Workbench",
  "description": "Compact density, flat cards, collections on the right. For developers.",
  "density": "compact",
  "variant": "flat",
  "features": {
    "stickyActionBar": false,
    "collapsibleSidebar": true,
    "brandedLogin": false,
    "highContrastMode": false,
    "reducedMotion": false,
    "layoutRegions": {
      "collections": "right",
      "mediaGalleries": "left"
    }
  },
  "layoutState": {
    "leftSidebar": "full",
    "rightSidebar": "full",
    "pageheader": "hidden",
    "pagefooter": "hidden",
    "header": "hidden",
    "footer": "hidden"
  },
  "customCss": "/* optional custom CSS */",
  "presetSource": "sveltycms-builtin"
}
```text

**Sync flow**: File save → Vite detects → `sveltyCmsPlugin.handleHmr()` → `adminThemeService.create/update` → DB → `server.ws.send("svelty:theme-update")` → client refreshes theme list.

## 7. Pillar D: Svelte 5 Snippet-Based Sticky Action Bar

The `<StickyActions>` wrapper component (`src/components/ui/sticky-actions.svelte`) uses Svelte 5's snippet pattern to register page action buttons into the layout's sticky bar:

```
<!-- Any page: wrap buttons in StickyActions -->
<StickyActions>
  <button onclick={save} aria-keyshortcuts="Mod+S">Save</button>
  <button onclick={reset}>Reset</button>
</StickyActions>
```text

**How it works:**

1. `<StickyActions>` renders buttons in-place AND sets `ui.stickyActionContent` to its children snippet
2. `+layout.svelte` renders `{@render ui.stickyActionContent()}` into a sticky bar when `theme.features.stickyActionBar` is true
3. On page destroy, `onMount` cleanup sets `ui.stickyActionContent = null`

**Accessibility**: `role="toolbar"`, `aria-label="Page actions"`, `aria-live="polite"`, native `<button>` elements with visible focus indicators. WCAG 2.2 AA compliant.

## 8. Visual Regression Prevention

1. **CSS Variable Fallbacks**: All `var(--admin-*, fallback)` references include the current value as fallback so the browser never renders a broken layout if no custom theme is served.
2. **Default Root Mappings**: `:root` in `utilities.css` initializes all `--admin-*` tokens to the current production values.
3. **Svelte 5 Context Fallback Guard**: Components resolve to cozy/bordered defaults when `getThemeContext()` returns `undefined`.
4. **Playwright Visual Regression**: `visual-regression` project with 7 screenshot baselines (`admin-theme/visual-regression.spec.ts`); CI-gated in `e2e-app` matrix.

## 10. Adaptive Workspaces

Beyond visual theming — SveltyCMS adapts the entire workspace to the user's role:

- **Role-Based Layouts**: Translators → localization tables focus; Managers → dashboard metrics; Developers → compact density + raw JSON views
- **Density Modes**: `compact` (heavy listings, developer logs), `cozy` (standard editing), `spacious` (distraction-free writer mode)
- **Real-Time Contrast Validation**: Dynamic color scale analysis verifying focus rings, text, and alert states meet WCAG 3.0 contrast ratios
- **Status-Color Tokens**: `--admin-color-draft`, `--admin-color-published`, `--admin-color-scheduled`, `--admin-color-review` linked directly to content review states
- **6-Region Layout Shell**: Each region independently toggleable via `ui-store`, persisted to DB per-theme, responsive across 6 breakpoints
- **Svelte 5 Snippet Pattern**: `<StickyActions>` demonstrates the pattern — any page wraps its actions and they appear in the sticky bar automatically
````

## 6. Default Theme & Extensibility

SveltyCMS ships with a single built-in admin theme: **Default** (`src/themes/default.json`). It provides a balanced starting point:

| Setting           | Value                             |
| ----------------- | --------------------------------- |
| Density           | `cozy` (via `--admin-density: 1`) |
| Card variant      | `bordered`                        |
| Branded login     | Enabled                           |
| Sticky action bar | Enabled                           |
| Primary accent    | Teal (#0f766e)                    |

### Adding Custom Theme Presets

To add additional admin theme variants, create new JSON files in `src/themes/`. They are auto-synced to the database on every server boot via `theme-file-sync.ts`.

**Example** — `src/themes/compact.json`:

```json
{
  "name": "Compact",
  "description": "High-density variant for power users",
  "density": "compact",
  "variant": "flat",
  "features": {
    "stickyActionBar": true,
    "collapsibleSidebar": true,
    "brandedLogin": false
  },
  "properties": {
    "primary": "#1d4ed8",
    "surface": "#f1f5f9"
  },
  "presetSource": "custom"
}
```

Admins can also create, clone, and customize themes entirely through the UI at **Config → Appearance** — no file editing required.

### Density-Only Toggling

Most admins only need density adjustments, which are controlled via `data-density` attribute (compact / cozy / spacious) — no separate theme file needed. These are configured per-theme in the Appearance panel.

### Component Usage Status

**AdminPageShell** and **AdminCard** components exist and are in active use:

- **AdminPageShell**: Used in `(app)/config/+page.svelte`, `(app)/config/access-management/+page.svelte`, and `(app)/config/appearance/+page.svelte`. Expansion to remaining `(app)` routes is in progress.
- **AdminCard**: Used across config routes (access management, appearance, design system) and the welcome theme picker. Replaces bare `<div class="card">` patterns incrementally.

Both components enforce the structural blueprint from `style-guide-gui.mdx`. See the CI gate `lint:admin-theme` for enforcement coverage.

### Media Details Inline Editing

The media details modal (`media-details-modal.svelte`) now supports **inline editing** of asset metadata: users can click-to-edit the asset name, alt text, and caption directly in the media drawer without opening a separate edit form. This aligns with the admin theme's goal of reducing modal friction and keeping context visible during editing.

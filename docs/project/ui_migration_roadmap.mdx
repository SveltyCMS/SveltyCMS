# Native Svelte 5 Component Library

## Goal

Replace heavy third-party UI libraries (Skeleton v4, Zag.js) with lightweight, native Svelte 5 components to eliminate the "ESM request storm," reduce bundle size, and improve performance while achieving **WCAG 3.0 Functional Performance** compliance — exceeding WCAG 2.2 AA.

---

## Current Status: 37 Components — Complete ✅

All 32 original Skeleton component types have a native Svelte 5 counterpart. **5 additional enterprise components** (Checkbox, RadioGroup, Select, Loader, RadioItem) were created that have no Skeleton equivalent. Zero production code imports from `@skeletonlabs/skeleton-svelte`. All 44 files have standard `@file`/`@component` headers.

---

## WCAG 3.0 Compliance Features

All components include:

- **`prefers-reduced-motion`** — 7 components with animation (checkbox, collapsible, loader, progress, radio-group, radio-item, tree-view)
- **24px minimum touch targets** — checkbox, radio-item, tree-view, toggle, button
- **Focus indicators with 3:1 minimum contrast** — `focus-visible:ring` on all interactive elements
- **`role="status"` live regions** — loader, tree-view, toast
- **`crypto.randomUUID()`** for generated IDs — accordion-item, collapsible, input, segmented-control (with `Math.random()` fallback in others for resilience)
- **RTL support** — `dir` prop on tree-view, chevron rotation on tabs

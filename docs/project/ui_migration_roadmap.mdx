# Roadmap: Native Svelte 5 Component Migration

## Goal

Replace heavy third-party UI libraries (Skeleton v4, Zag.js) with lightweight, native Svelte 5 components to eliminate the "ESM request storm," reduce bundle size, and improve performance while maintaining **WCAG 2.2 AA** accessibility.

---

## Current Status: 32/32 Skeleton Components Replaced ✅

All 32 Skeleton components have a native Svelte 5 counterpart in `src/components/ui/`. Zero production code imports from `@skeletonlabs/skeleton-svelte`. The only remaining Skeleton reference is the ui-test comparison lab, which imports both libraries for side-by-side validation.

## Phase 1: Core Primitives (Complete) ✅

**Status**: All core primitives implemented and verified in the Kitchen Sink lab.

- `Button`, `Input`, `Toggle`, `Card`, `Badge`, `Avatar`
- **Technology**: Svelte 5 Runes (`$state`, `$props`), Tailwind 4 CSS variables
- **Architecture**: Logic decoupled from visuals — components remain functional with any theme swap

## Phase 2: Accessibility as Actions (Complete) ✅

Replaced Zag.js state machines with lightweight Svelte `use:action` patterns:

- `focusTrap` — modals and drawers
- `clickOutside` — dropdowns, popovers
- `ariaAnnounce` — live regions
- `keyboardNav` — tab/arrow key handling

## Phase 3: Theming & Design System (Complete) ✅

Migrated from Skeleton design tokens to Tailwind 4 CSS variables:

- Centralized `app.css` with `[data-theme="sveltycms"]` block
- Swappable corporate themes via the Skeleton theme generator
- Dark mode via `media` and `class` selectors
- CSS utilities extracted to `src/utilities.css` (gradients, wrappers, a11y)

## Phase 4: Complex Layout & Data Components (Complete) ✅

1:1 functional parity achieved for all interactive and data components:

- **Layout**: `Modal`, `Drawer` (native `<dialog>` + Svelte 5)
- **Interactive**: `Accordion`, `Collapsible`, `Stepper`, `Slider`, `Rating`, `FileUpload`, `StatusBadge`, `Tags`
- **Data**: `Table`, `TreeView`, `Breadcrumb`
- **Inputs**: `Combobox`, `DatePicker`, `FloatingInput`, `SegmentedControl`

## Phase 5: Kitchen Sink Validation Lab (Active)

**Source**: `src/routes/ui-test/+page.svelte`

All 32 native components are compared 1:1 against their Skeleton equivalents in a dark-mode toggleable side-by-side layout. The lab verifies:

- Visual parity between native and Skeleton components
- Dark/light mode correctness
- Keyboard navigation and focus management
- Screen reader accessibility
- TypeScript type safety (`bun run check` passes)

**Validation command**: `bun run check`

## Expected Outcomes

| Metric                          | Before (Skeleton)     | After (Native)   |
| ------------------------------- | --------------------- | ---------------- |
| **TTFB / Cold Start**           | ~200ms                | <100ms           |
| **Dev Network Requests**        | ~1,500                | <150             |
| **Production JS Weight**        | ~850 KB (Brotli)      | ~500 KB (Brotli) |
| **Build Time**                  | ~14s                  | ~10s (target)    |
| **WCAG 2.2 AA**                 | ✅                    | ✅               |
| **Third-party UI Dependencies** | 2 (Skeleton + Zag.js) | 0                |

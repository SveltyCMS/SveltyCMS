# docs.sveltycms.com — Implementation Plan

The docs site imports `docs/` content 1:1. No content changes required. All 7 competitive features are implemented as **shell components** that wrap the rendered MDX.

---

## 1. Architecture: Content-Agnostic Shell

```
docs.sveltycms.com (SvelteKit app)
│
├── reads docs/ at build time ──→ no content modification needed
├── parses frontmatter ─────────→ title, description, order
├── converts MDX to HTML ───────→ headings, code blocks, callouts, tables, mermaid
└── wraps in shell components ──→ sidebar, TOC, edit link, feedback, search
```

> **Zero content changes**: Every `.mdx` file in `docs/` renders as-is. The shell adds navigation, search, and UX features around the content.

## 4. MDX → HTML Rendering (Server-Side)

No runtime MDX parser dependency needed. Simple regex converter in `+page.server.ts`:

| MDX Feature              | Converter                                        |
| ------------------------ | ------------------------------------------------ |
| Frontmatter              | `---\n...\n---` → stripped, parsed into metadata |
| `# Heading`              | → `<h1 id="slug">`                               |
| `## Heading`             | → `<h2 id="slug">`                               |
| `### Heading`            | → `<h3 id="slug">`                               |
| `` `code` ``             | → `<code>`                                       |
| ` ```lang\ncode\n``` `   | → `<pre><code class="language-lang">`            |
| ` ```mermaid\n...\n``` ` | → `<div class="mermaid">`                        |
| `[text](url)`            | → `<a href="url">`                               |
| `![alt](url)`            | → `<img src="url" alt="alt" loading="lazy">`     |
| `**bold**`               | → `<strong>`                                     |
| `*italic*`               | → `<em>`                                         |
| `> [!NOTE]`              | → styled `<blockquote>` with icon                |
| `> [!TIP]`               | → styled `<blockquote>` with icon                |
| `> [!IMPORTANT]`         | → styled `<blockquote>` with icon                |
| `> [!WARNING]`           | → styled `<blockquote>` with icon                |
| `> [!CAUTION]`           | → styled `<blockquote>` with icon                |
| `\| table \|`            | → `<table><tr><th>` or `<td>`                    |
| `---`                    | → `<hr>`                                         |

## 5. Sidebar Generation

`+page.server.ts` scans `docs/` directory tree at request time:

```
docs/
├── getting-started.mdx    → Getting Started
├── README.mdx             → Documentation Hub
├── troubleshooting.mdx    → Troubleshooting
├── api/                   → API (collapsible group)
│   ├── index.mdx
│   ├── auth.mdx
│   └── ...
├── architecture/          → Architecture (collapsible group)
│   ├── index.mdx
│   ├── security/          → Security (nested group)
│   └── ...
├── guides/                → Guides (collapsible group)
├── contributing/          → Contributing (collapsible group)
├── project/               → Project (collapsible group)
└── tests/                 → Tests (collapsible group)
```

- Items sorted by `order` field in frontmatter (if present), then alphabetically
- Directories become collapsible `<details>` groups
- Current page highlighted with accent color
- Active section auto-expanded

## 6. Data Flow

```
Request: GET /docs/architecture/security/index
  │
  ▼
+page.server.ts
  │
  ├─ Read ../../docs/architecture/security/index.mdx
  ├─ Parse frontmatter → { title, description, order, updated }
  ├─ Convert body to HTML
  ├─ Build sidebar tree (one-time, cached)
  ├─ Build breadcrumbs from URL segments
  ├─ Find prev/next from flat sidebar
  ├─ Compute reading time
  └─ Return { title, description, html, sidebar, breadcrumbs, prevPage, nextPage, readingTime, version }
  │
  ▼
+page.svelte
  │
  ├─ Render sidebar
  ├─ Show version · reading time · edit link
  ├─ Show breadcrumbs
  ├─ {@html data.html} (the MDX content)
  ├─ Inject code copy buttons via onMount
  ├─ Show feedback widget
  └─ Show prev/next navigation
```

## 8. Non-Negotiables

> [!IMPORTANT]
> These must be true for launch:

- [ ] **Zero changes to `docs/` content** — the site must render existing MDX files as-is
- [ ] **No YAML dependency** — parse frontmatter with regex or colon-delimited parser
- [ ] **No MDX compiler at runtime** — simple regex-based HTML conversion is sufficient and fast
- [ ] **All links relative to docs/** — the site resolves `./page.mdx` and `../other/page.mdx` correctly
- [ ] **Mermaid rendered client-side** — inject `<div class="mermaid">` and let mermaid.js hydrate in browser
- [ ] **EU-compliant** — all competitive claims already qualified in source content
- [ ] **0 lint errors** — the docs/ linter passes on every commit

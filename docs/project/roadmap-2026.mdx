# SveltyCMS: 2026 Todo List

> [!NOTE]
> **EU Compliance**: Competitive comparisons in this document are based on publicly available documentation as of June 2026. Feature availability claims reflect our own implementation status and publicly documented features of referenced platforms.

This document outlines the remaining high-priority tasks and strategic objectives for SveltyCMS in 2026.

## ✅ Recently Completed (May–June 2026)

- ✅ **Native UI Component Library v2**: 11 components enhanced with zero new dependencies. `useFloating` rune replaces `@floating-ui/dom` with CSS Anchor Positioning + JS fallback. `useDialog` rune adds focus trapping and WCAG 3.0 ARIA to modals/drawers. Combobox gains clearable, groupBy, debounce, loading. Rating gains fractional values. Table gains virtual scrolling. Toast gains swipe dismiss, richColors, callbacks, Alt+T hotkey. DatePicker gains datetime-local/time support.
- ✅ **Stripe Payments Plugin**: Full plugin with PaymentIntent creation, webhook handler, PCI-compliant Stripe Elements form, tenant-level API keys. Uses official Stripe SDK via dynamic import (zero overhead when disabled).
- ✅ **Plugin Documentation**: MDX docs for all 7 existing plugins + Stripe. Plugin architecture doc rewritten with current API, config-page pattern, sandbox boundaries, and Stripe case study.
- ✅ **Dependency Reduction**: Removed `@floating-ui/dom` (3 packages). Zero external positioning dependencies. Compositor-level performance for Chrome/Firefox users.
- ✅ **WCAG 3.0**: 16 accessibility fixes across popover, tooltip, modal, drawer, table, toast. Focus trapping, aria-modal, aria-haspopup, aria-expanded, Escape key, aria-sort.
- ✅ **June 2026 Enhancement Sprint — Full Feature Parity**: Comprehensive sprint closing every remaining competitive gap:
  - **Adaptive Job Scheduler** (`src/services/scheduler.ts`): Background job runner with 1s–30s adaptive polling, exponential retries, and crypto-chained audit integration. Scheduled publishing now executes — not just stores intent.
  - **Per-Field Content Localization**: `Record<Locale, string>` field storage with inline per-field locale switchers and one-click AI translation via `src/services/ai-translation.ts`. Legacy string fields auto-migrate.
  - **Crypto-Chained Audit Logs** (`src/services/audit-chain.ts`): SHA-256 hash chain across all audit entries with `previousHash`/`chainHash` columns, chain verification, and per-content audit viewer component.
  - **Zero-Dependency Full-Text Search** (`src/services/search-service.ts`): Database-native FTS across PostgreSQL (`tsvector`/`tsquery`), MariaDB (`MATCH...AGAINST`), SQLite (FTS5 BM25), and MongoDB (`$text`). Zero npm dependencies.
  - **AI-Smart CMS Importer** (`src/services/smart-importer.ts`): 5-format auto-detection (WordPress WXR, Strapi, Directus, Drupal, SveltyCMS), 30+ heuristic field mappings, ACF/CMB2 detection, batch processing, drag-and-drop UI.
  - **Quick-Start Collection Templates** (`src/routes/setup/presets.ts`): 5 pre-built schemas (Blog, Agency, SaaS, Corporate, E-commerce) with full field definitions and widget types.
  - **Progressive Initialization UX** (`src/routes/warming-up/`): SSR-rendered warming-up dashboard with real-time service status replacing 503 errors during system boot.
  - ✅ **Collection Builder Enhancements**: Template selection modal, hover effects, descriptive tooltips on all field configuration options.
  - ✅ **June 2026 Comprehensive Security Audit**: 6 gaps closed — SVG polyglot sanitization, S3 endpoint SSRF prevention, Ollama RAG prompt injection defense, GraphQL introspection blocking, Tiptap link protocol hardening, rate-limiter dependency hygiene. Full documentation at [Security Docs](/docs/architecture/security/index.mdx).
  - ✅ **June 2026 Security Hardening**: Full audit remediation — 16 code fixes, account lockout wired to login path (5 attempts → 15-min lock, HTTP 423), GraphQL depth (7) + alias (15) limits via envelop plugin, MongoDB NoSQL injection protection (`sanitizeMongoQuery` blocks `$where`/`$function`/`$expr`), session rotation 60→15 min, SAML CSRF dual-cookie protection, soft-delete filter enforcement, bulk operation size limits (1000 items), email endpoint wired to `sendMail()`, port validation (1–65535).
  - ✅ **Enterprise Content System**: Schema field validation (duplicate `db_fieldName` detection), hardened `isSafeCollectionPath` (path traversal + special char blocking), incremental change-set tracking (`_changedFiles` Set), native `worker_threads` module pool (`module-worker-pool.server.ts`), content plugin system (`onSchemaLoad`/`onReconcile`/`onNodeChange` hooks), lightweight performance tracing (`traceStart`/`getTraceStats`).
  - ✅ **Enterprise Security Audit v2.0**: OWASP-mapped automated auditor with authenticated testing (`--auth`), GraphQL DoS detection, NoSQL injection probes, cookie security attribute checks, CORS preflight verification, timing side-channel detection.

## 🚀 Phase 5: Secure Upgrade & Cloud Strategy

- 🏗️ **Phase 2: Hybrid Package Model** — _In Progress_: Scoped packages `packages/core` (`@sveltycms/core`) and `packages/widgets` (`@sveltycms/widgets`) created with barrel exports to existing `src/`. Monorepo workspace configured. Pending: npm publishing pipeline, CI integration, consumer documentation.
- 🚀 **Phase 3: Intelligence & Rollbacks**: Automated visual regression diffs before upgrades and one-click database/code rollbacks.
  - 🏗️ **Version Channels (LTS, Stable, Next)** — _In Progress_: Core service (`src/services/core/version-service.ts`) and API endpoints (`/api/version/channels`, `/api/version/check`) implemented. Pending: UI dashboard integration and upgrade CLI channel selection.
- 🚀 **Svelty Managed Cloud**: Optional managed Postgres + Redis tier (Fly.io/Cloudflare) for zero-ops enterprise deployments.

## ✅ Gap Analysis (Resolved June 2026)

1. ✅ **Deep Draft/Versioning Integration**: Per-item content revisions with diff, restore, auto-cleanup. Revisions tab in content editor with full history browsing and comparison.
2. ✅ **Nested UI Logic**: Nested drag-and-drop treeview stabilized. Collection Builder templates provide instant onboarding for complex hierarchies.
3. ✅ **Scheduled Publishing Execution**: Job scheduler now processes scheduled status transitions, eliminating the previous vaporware gap.
4. ✅ **Full-Text Search**: Zero-dependency database-native FTS deployed across all adapters.

## 🚀 Competitive Leapfrog Opportunities

Strategic features designed to push SveltyCMS ahead of the 2026 market:

1. 🏗️ **Autonomous Schema-Aware AI Co-Pilot** — _In Progress_: `@sveltycms/ai` package created with `suggestFields()` (schema suggestions), `scoreContent()` (quality scoring), and `suggestMapping()` (import field mapping). Core AIService extended with Co-Pilot methods. Pending: MCP client-side UI block spawning, Ghost data structure generation from Valibot schemas.
2. ✅ **Bulk AI Content Localization**: Per-field AI translation implemented — background service for high-volume automated translation of existing entries while respecting brand-voice guidelines.
3. **AI-Verified Zero-Downtime Deployments**: Self-healing pipelines with automated visual regression and performance audits (Lighthouse CI) before final production promote.
4. **Autonomous Import Optimization**: Dynamic concurrency and TTL-backed background imports for 30k+ row datasets.
5. **"Time-Travel" Git-Style Content Branching**: Multi-collection branching for campaign management (e.g., "Black Friday Sale") with visual diffs and mathematical merge conflict resolution.
6. **Decentralized WebGPU Media Engine**: Client-side WASM/WebGPU image processing and focal-crop generation to reduce server load and improve edge performance.
7. **Dynamic Administration Theme System**: Decouple layout structural options (e.g., Drupal Gin-level flexibility) from colors via Svelte 5 contextual design tokens (spacing, borders, shadows, layout templates).
8. **Unified WASM Formatting Pipeline**: Evaluate [tsv.dev](https://www.tsv.dev/) as a next-generation unified formatter and parser for Svelte, TypeScript, and CSS to eventually replace Prettier and streamline our development toolchain.

## 🏢 Enterprise Workspaces & Governance (Roadmap Additions)

We are introducing a fourth framing layer focusing on dynamic collaboration, security traceability, and workflow control:

1. **"Whole-Team Studio" Workflow Pillar**: Dedicated framing layers for editors, reviewers, translators, and managers to transition SveltyCMS from a developer-focused tool to an enterprise content workspace.
2. **Unbending Editorial State Visibility**: Unify draft, published, scheduled, and review states across all entry lists, editor headers, sidebars, bulk actions, live previews, and global search.
3. **Translation Cockpit**: Expand smart translation to support collection-level translation targets, shared translation glossaries, brand tone/style parameters, review status matrices, a missing-locale mapping grid, and human-in-the-loop approval gates.
4. **Adaptive Workspaces Theme System**: Evolve the branding plan to role-based dashboard layouts, custom density levels (compact, cozy, spacious), client theme previews, contrast validation (WCAG 3.0/APCA), and per-workflow dashboards.
5. **OAuth 2.1 Governance for MCP**: Dynamic client registration, scoped permissions, user consent dialogs, detailed audit logs, token expiry/revocation, and "agent acted as user X" trace logging for AI agent operations.
6. **Deep JSON/Object Field Filtering**: Implement path-based SQL/NoSQL filtering for JSON and object widget fields across REST, GraphQL, and LocalCMS, backed by high-fidelity performance benchmarks.
7. **AI Command Bar Guardrails**: Permission-aware command gating, change preview panels, interactive diff rendering, allowed/blocked action explanations, and comprehensive audit logs.

# SveltyCMS: 2026 Todo List

This document outlines the remaining high-priority tasks and strategic objectives for SveltyCMS in 2026.

## 🚀 Phase 5: Secure Upgrade & Cloud Strategy (Q2-Q4 2026)

- 🚀 **Phase 2: Hybrid Package Model (Q2 2026)**: Publishing core modules as scoped packages (`@sveltycms/core`, `@sveltycms/widgets`) while maintaining Git-based "Agency OS" flexibility.
- 🚀 **Phase 3: Intelligence & Rollbacks (Q3 2026)**: Support for version channels (LTS, Stable, Next), automated visual regression diffs before upgrades, and one-click database/code rollbacks.
- 🚀 **Svelty Managed Cloud (Q4 2026)**: Optional managed Postgres + Redis tier (Fly.io/Cloudflare) for zero-ops enterprise deployments.

## 🎯 Gap Analysis (Priority Todo)

These items are required to achieve total feature parity with enterprise competitors:

1. **Deep Draft/Versioning Integration**: Maturing the "Enterprise Handshake Protocol" to seamlessly bind Live Preview with complex document revision histories.
2. **Nested UI Logic**: Stabilizing the nested drag-and-drop treeview layer for ultra-complex hierarchy management without UI flicker.

## 🚀 Competitive Leapfrog Opportunities

Strategic features designed to push SveltyCMS ahead of the 2026 market:

1. **Autonomous Schema-Aware AI Co-Pilot**: Client-side MCP AI that autonomously spawns UI blocks and "Ghost" data structures based on Valibot schemas.
2. **Bulk AI Content Localization**: Background service for high-volume automated translation of existing entries while respecting brand-voice guidelines.
3. **AI-Verified Zero-Downtime Deployments**: Self-healing pipelines with automated visual regression and performance audits (Lighthouse CI) before final production promote.
4. **Autonomous Import Optimization**: Dynamic concurrency and TTL-backed background imports for 30k+ row datasets.
5. **"Time-Travel" Git-Style Content Branching**: Multi-collection branching for campaign management (e.g., "Black Friday Sale") with visual diffs and mathematical merge conflict resolution.
6. **Decentralized WebGPU Media Engine**: Client-side WASM/WebGPU image processing and focal-crop generation to reduce server load and improve edge performance.

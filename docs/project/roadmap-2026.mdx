# SveltyCMS: 2026 Todo List

This document outlines the remaining high-priority tasks and strategic objectives for SveltyCMS in 2026.

## ✅ Recently Completed (May 2026)

- ✅ **Native UI Component Library v2**: 11 components enhanced with zero new dependencies. `useFloating` rune replaces `@floating-ui/dom` with CSS Anchor Positioning + JS fallback. `useDialog` rune adds focus trapping and WCAG 3.0 ARIA to modals/drawers. Combobox gains clearable, groupBy, debounce, loading. Rating gains fractional values. Table gains virtual scrolling. Toast gains swipe dismiss, richColors, callbacks, Alt+T hotkey. DatePicker gains datetime-local/time support.
- ✅ **Stripe Payments Plugin**: Full plugin with PaymentIntent creation, webhook handler, PCI-compliant Stripe Elements form, tenant-level API keys. Uses official Stripe SDK via dynamic import (zero overhead when disabled).
- ✅ **Plugin Documentation**: MDX docs for all 7 existing plugins + Stripe. Plugin architecture doc rewritten with current API, config-page pattern, sandbox boundaries, and Stripe case study.
- ✅ **Dependency Reduction**: Removed `@floating-ui/dom` (3 packages). Zero external positioning dependencies. Compositor-level performance for Chrome/Firefox users.
- ✅ **WCAG 3.0**: 16 accessibility fixes across popover, tooltip, modal, drawer, table, toast. Focus trapping, aria-modal, aria-haspopup, aria-expanded, Escape key, aria-sort.

## 🚀 Phase 5: Secure Upgrade & Cloud Strategy

- 🏗️ **Phase 2: Hybrid Package Model** — _In Progress_: Scoped packages `packages/core` (`@sveltycms/core`) and `packages/widgets` (`@sveltycms/widgets`) created with barrel exports to existing `src/`. Monorepo workspace configured. Pending: npm publishing pipeline, CI integration, consumer documentation.
- 🚀 **Phase 3: Intelligence & Rollbacks**: Automated visual regression diffs before upgrades and one-click database/code rollbacks.
  - 🏗️ **Version Channels (LTS, Stable, Next)** — _In Progress_: Core service (`src/services/core/version-service.ts`) and API endpoints (`/api/version/channels`, `/api/version/check`) implemented. Pending: UI dashboard integration and upgrade CLI channel selection.
- 🚀 **Svelty Managed Cloud**: Optional managed Postgres + Redis tier (Fly.io/Cloudflare) for zero-ops enterprise deployments.

## 🎯 Gap Analysis (Priority Todo)

These items are required to achieve total feature parity with enterprise competitors:

1. **Deep Draft/Versioning Integration**: Maturing the "Enterprise Handshake Protocol" to seamlessly bind Live Preview with complex document revision histories.
2. **Nested UI Logic**: Stabilizing the nested drag-and-drop treeview layer for ultra-complex hierarchy management without UI flicker.

## 🚀 Competitive Leapfrog Opportunities

Strategic features designed to push SveltyCMS ahead of the 2026 market:

1. 🏗️ **Autonomous Schema-Aware AI Co-Pilot** — _In Progress_: `@sveltycms/ai` package created with `suggestFields()` (schema suggestions), `scoreContent()` (quality scoring), and `suggestMapping()` (import field mapping). Core AIService extended with Co-Pilot methods. Pending: MCP client-side UI block spawning, Ghost data structure generation from Valibot schemas.
2. **Bulk AI Content Localization**: Background service for high-volume automated translation of existing entries while respecting brand-voice guidelines.
3. **AI-Verified Zero-Downtime Deployments**: Self-healing pipelines with automated visual regression and performance audits (Lighthouse CI) before final production promote.
4. **Autonomous Import Optimization**: Dynamic concurrency and TTL-backed background imports for 30k+ row datasets.
5. **"Time-Travel" Git-Style Content Branching**: Multi-collection branching for campaign management (e.g., "Black Friday Sale") with visual diffs and mathematical merge conflict resolution.
6. **Decentralized WebGPU Media Engine**: Client-side WASM/WebGPU image processing and focal-crop generation to reduce server load and improve edge performance.
7. **Dynamic Administration Theme System**: Decouple layout structural options (e.g., Drupal Gin-level flexibility) from colors via Svelte 5 contextual design tokens (spacing, borders, shadows, layout templates).

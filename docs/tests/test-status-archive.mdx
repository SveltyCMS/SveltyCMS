**Hardening applied**:

- Security rate-limiter reset between tests тЖТ zero cross-test leakage
- `IDLE` state accepted in health checks тЖТ eliminates 10s settle delay
- Widget auto-registration тЖТ fixes 404 for valid widgets in fresh DB
- All caches + state stores cleared per-test тЖТ true isolation
- **Test DB safety guard** (`scripts/check-test-db-safety.ts`) runs in pre-commit and pre-push hooks тЖТ blocks unsafe `config/private.test.ts` before tests start
- **Shared classifier** (`src/utils/test-db-safety.ts`) used by server, integration runner, and git hooks тЖТ single source of truth for isolated DB name validation
- **Secret synchronization** тАФ integration runner writes `TEST_API_SECRET` to `tests/e2e/.auth/test-secret.txt` before server start тЖТ eliminates 401 mismatch
- **`SKIP_COLLECTION_COMPILE`** flag prevents @src alias resolution failures during integration test builds on Windows
- **Multi-DB contract test** (`contract.test.ts`) тАФ 25 tests across 6 security gates (Auth, Collection CRUD, User Batch, Setup Gating, Fail-Closed, Media Permissions) running on all 4 adapters
- **CI scorecard** тАФ per-DB markdown table + JSON summary written to `GITHUB_STEP_SUMMARY` in `run-integration-tests.ts`; each adapter run emits pass/fail/failures for every spec

These measures ensure zero cross-test contamination and deterministic test results.

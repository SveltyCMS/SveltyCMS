# Dashboard (`/dashboard`) ‚Äî Control Map

> [!IMPORTANT]
> **Widgets are install-specific.** Core ships a default set under `src/routes/(app)/dashboard/widgets/`; plugins may inject more via slots or additional files.  
> Tests and docs assert **shell contracts** and **API shapes**, never ‚Äúmust show CPU + Memory + Logs‚Äù.

## Architecture (brief)

| Piece                             | Role                                                                            |
| --------------------------------- | ------------------------------------------------------------------------------- |
| `+page.server.ts`                 | Auth, `dashboard:read` gate, compile-time widget discovery (`import.meta.glob`) |
| `+page.svelte`                    | Toolbar, add/search/reset, grid, DnD reorder, lazy load, AI generative mode     |
| `base-widget.svelte`              | Shared fetch/poll/cache/refresh chrome for individual widgets                   |
| `dashboard-preferences.svelte.ts` | Persist layout to `/api/system-preferences`                                     |
| `widgets/*.svelte`                | **Per-install catalog** ‚Äî each exports `widgetMeta`                             |
| `Slot name="dashboard"`           | Plugin injection zone                                                           |

## Control ‚Üí test matrix (shell = 100%)

### Page shell

| Control               | Unit            | Integration | E2E                 | Notes                        |
| --------------------- | --------------- | ----------- | ------------------- | ---------------------------- |
| Load + title          | page.server     | ‚Äî           | ‚úÖ `dashboard.spec` | AdminPageShell               |
| `dashboard:read` deny | page.server 403 | ‚Äî           | RBAC suite          | Non-admin without perm       |
| Empty state           | ‚Äî               | ‚Äî           | ‚úÖ                  | Install with no saved layout |
| Widget grid           | ‚Äî               | ‚Äî           | ‚úÖ                  | When preferences non-empty   |
| Plugin slot attached  | ‚Äî               | ‚Äî           | ‚úÖ                  | May be empty                 |

### Toolbar (install-agnostic)

| Control                           | Unit | E2E                                         |
| --------------------------------- | ---- | ------------------------------------------- |
| Add Widget button / menu          | ‚Äî    | ‚úÖ opens menu when catalog has free widgets |
| Widget search                     | ‚Äî    | ‚úÖ nonsense query ‚Üí ‚ÄúNo widgets found‚Äù      |
| Reset all                         | ‚Äî    | ‚úÖ after add ‚Üí empty state                  |
| AI toggle present                 | ‚Äî    | ‚úÖ does not require live MCP                |
| Pointer drag-reorder (‚â•2 widgets) | ‚Äî    | ‚úÖ header-band mouse drag                   |
| Keyboard Ctrl+Arrow reorder       | ‚Äî    | ‚úÖ accessibility path                       |

### Layout persistence

| Control           | Unit / integration                             | E2E                       |
| ----------------- | ---------------------------------------------- | ------------------------- |
| Save layout POST  | system-preferences API (integration elsewhere) | add+reset exercises store |
| CSRF on save      | preferences store sends header                 | covered by store code     |
| Secure widget ids | `generateUUID` (no Math.random)                | code review + unit path   |

### Per-widget content

| Policy                         | How we test                                                          |
| ------------------------------ | -------------------------------------------------------------------- |
| **Do not** assert widget names | E2E only counts `[data-widget-id]`                                   |
| **Do** test APIs widgets call  | `tests/integration/api/dashboard.test.ts` (health, metrics, logs, ‚Ä¶) |
| Optional deep unit per widget  | Opt-in; not required for shell 100%                                  |

### Dashboard API (shared by many widgets)

Covered by `tests/integration/api/dashboard.test.ts`:

- `GET /api/dashboard/health`
- `GET /api/dashboard/metrics` (+ detailed)
- system-info, logs, last5-content/media, online-user, system-messages, cache-metrics

# Bundle Size Analysis & Performance Metrics (April 2026)

We actively monitor our application bundle size and API latency to prevent performance regressions. Our target is **sub-10ms database persistence** and **sub-1s cold starts**.

## 📊 Enterprise Performance Matrix

The **Enterprise Benchmark Matrix** is our primary tool for cross-database performance validation. It measures:

- **REST API Latency**: E2E collection throughput.
- **GraphQL Resolvers**: Efficiency and N+1 resolution.
- **Relational Performance**: JOINs, population, and nested relations (Depth 2-3).
- **Middleware Overhead**: Latency added by Security, Auth, and RBAC layers.
- **Cold Start Time**: Server initialization latency.

To run the full matrix across all supported databases:

```bash
bun run scripts/enterprise-matrix.ts
```

> [!NOTE]
> This requires running Docker instances of MongoDB, PostgreSQL, and MariaDB.

## 🔗 Relational Performance Testing

With the 2026 update, we've introduced specialized relational benchmarks to ensure that complex data structures remain performant:

- **GQL Nested**: Tests Depth 2-3 relationship resolution.
- **REST Relational Search**: Validates filtering by related fields via the aggregation engine.

**File**: `tests/benchmarks/relational-performance.test.ts`

## Running Analysis Locally

To check the bundle size and basic performance locally:

```bash
bun run build:stats
```

This generates detailed reports including:

- Size of each chunk (raw, gzipped, brotli).
- Comparison with the previous build size.
- Warnings reported if predefined performance budgets are exceeded.
- An exported `bundle-report.json` designed for automated CI consumption.

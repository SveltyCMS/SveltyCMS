# Security Testing

**Last Updated:** April 26, 2026

SveltyCMS implements rigorous security testing to ensure the highest level of protection. All security tests follow the **no-backdoor policy** - tests must pass through real security guards.

## Test Categories

### 1. Encryption & Cryptography

| Test Suite                 | Tests | Status     |
| -------------------------- | ----- | ---------- |
| AES-256-GCM Encryption     | 14    | ✅ Passing |
| Argon2id Password Hashing  | 6     | ✅ Passing |
| Audit SHA-256 Chaining     | 12    | ✅ Passing |
| Preview HMAC (SHA-256)     | 4     | ✅ Passing |

**Key Breakthrough**: Cryptographic hashing for audit logs is now offloaded to a **Worker Thread Pool** to prevent main-thread latency spikes during high-volume ingestions.

### 2. Infrastructure Security (v2.0)

- ✅ **Fail-Closed Dispatcher**: The unified dispatcher ensures that unmapped routes are physically unreachable. 100% endpoint coverage.
- ✅ **Chaos Resilience**: Our security tests now include infrastructure "brownout" simulations to ensure that auth/RBAC logic never defaults to "Open" if the database is lagging.
- ✅ **Multi-Tenant Isolation**: Verified mathematical data separation via the `tenantId` adapter hook.

## Running Security Tests

```bash
# Run all security-related unit tests
bun run test:unit -- security

# Run the Audit Resilience Audit
# Measures system stability under infrastructure stress
bun test tests/benchmarks/chaos-resilience.test.ts
```

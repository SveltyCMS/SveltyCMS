# SveltyCMS Test Suite Status

> **Last Updated:** June 9, 2026
> **Source of Truth:** GitHub Actions and PR checks
> **Scope of This Page:** The maintained gates and safety rules, not hard-coded pass totals

## Executive Summary

SveltyCMS validates pull requests with a layered test pipeline:

- **Bootstrap / Build Gates:** install, lint, type-check, dependency audit, and production build
- **White-Box Unit Tests:** fast logic tests with Vitest
- **Black-Box Integration Tests:** real HTTP contract tests across SQLite, MongoDB, MariaDB, and PostgreSQL
- **Playwright E2E:** staged browser coverage for `wizard`, `auth-setup`, `signup`, `content`, and `system`

Do not treat documentation totals as the live status signal. The current branch verdict belongs to the latest workflow run.

## PR-Gating Suites

| Layer              | Job / Command                      | Purpose                                                  |
| ------------------ | ---------------------------------- | -------------------------------------------------------- |
| **Bootstrap**      | GitHub Actions `bootstrap`         | Lockfile install, workspace metadata, shared test secret |
| **Static Checks**  | `lint`, `type-check`, `build`      | Lint, compile, and production-build safety               |
| **Unit**           | `bun run test:unit`                | White-box logic and component-adjacent behavior          |
| **Integration**    | `bun run test:integration`         | Black-box HTTP contracts and DB parity                   |
| **E2E Wizard**     | Playwright `wizard`                | First-run provisioning and redirect contract             |
| **E2E Auth Setup** | Playwright `auth-setup`            | Auth bootstrap and reusable browser state                |
| **E2E App Smokes** | Playwright `signup/content/system` | Maintained browser smoke coverage for the admin UI       |

## Safety Rules

- `config/private.test.ts` is the source of truth for black-box and E2E jobs.
- Any temporary `config/private.ts` created by CI must mirror the test config only.
- `TEST_MODE=true` and `TEST_API_SECRET` are required for `/api/testing`.
- Local loopback should remain `127.0.0.1` for consistent cross-platform behavior.
- SQLite test databases must use isolated names and must never reuse active runtime data.

## Current Maintenance Policy

- Legacy exploratory Playwright specs may exist in `tests/e2e/`, but only the maintained smoke projects are PR-gating.
- Integration tests should validate behavior through HTTP and authenticated session cookies, not through privileged internal imports.
- When endpoint response shapes change, update the black-box tests and the route contract together.

## Local Commands

````bash
# White-box unit suite
bun run test:unit

# Black-box API suite
bun run test:integration

# Local Playwright convenience flow
bun run test:e2e

# Focused Playwright projects
bun x playwright test tests/e2e/setup-wizard.spec.ts --project=wizard
bun x playwright test --project=auth-setup
bun x playwright test --project=signup
bun x playwright test --project=content
bun x playwright test --project=system
```text

## Verify Live Status
````

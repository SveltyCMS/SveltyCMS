## 2. Health by layer

| Layer                       | Confidence         | What it proves                                                      | How to run                                                        |
| --------------------------- | ------------------ | ------------------------------------------------------------------- | ----------------------------------------------------------------- |
| **Unit** (Vitest)           | **High**           | Fast white-box: auth, security, widgets, content, data-ops, helpers | `bun run test:unit`                                               |
| **Security regression**     | **High**           | Hooks + RBAC defense-in-depth (~2s local)                           | See AGENTS.md security command                                    |
| **Integration** (black-box) | **High on SQLite** | Real HTTP + DB; multi-adapter in CI (Maria/Mongo/PG may differ)     | `bun run scripts/run-integration-tests.ts --db=sqlite --no-build` |
| **E2E** (Playwright)        | **Medium–High**    | Browser journeys; some suites still flaky/soft in places            | `bun x playwright test --project=chromium`                        |
| **Benchmarks**              | **Selective**      | Perf matrix; full suite is long — not every PR                      | See [benchmarks](../project/benchmarks/index.mdx)                 |

**Gates (local → CI)**

| When       | What runs (summary)                                                  |
| ---------- | -------------------------------------------------------------------- |
| Pre-commit | Format/lint/slop on staged + fast security set                       |
| Pre-push   | Full unit, build (when needed), optional DB tasks, P0 coverage check |
| CI         | Matrix: lint/types/unit → build → integration × DBs → E2E projects   |

## 5. What to run locally

```bash
# Daily / before commit
bun run gate:fast

# Before push (matches pre-push spirit)
bun run verify:push

# Unit only
bun run test:unit

# Integration (SQLite, no rebuild if already built)
bun run scripts/run-integration-tests.ts --db=sqlite --no-build

# E2E (example: user journeys)
bun x playwright test tests/e2e/routes/user/p0-journeys.spec.ts

# Full local CI-ish (incl. E2E — slow)
bun run ci:local
```

Safety: tests must never point at production DBs — see test-config safety guards in [git-workflow.mdx](./git-workflow.mdx).

## 7. How to keep this page honest

When you ship a meaningful test or stability change:

1. Update **this file** only if health, open gaps, or “hot areas” change (few sentences + table rows).
2. Put deep numbers, file inventories, and “resolved” write-ups in the **[archive](./test-status-archive.mdx)** (or route-specific docs).
3. Never treat static totals as CI truth — link to Actions.

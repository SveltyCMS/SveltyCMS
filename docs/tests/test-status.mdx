## ✅ Passing Tests (Latest Wins)

### 1. Database & Adapter Resilience 🗄️

- ✅ **Drizzle SQLite-Proxy Bridge**: Hardened query execution mapping when running Vitest on Node.js. Automatically wraps rows returned by Node's native `DatabaseSync` as arrays of values to perfectly align with Drizzle's `sqlite-proxy` contract, preventing dynamic schema-drift serialization failures.
- ✅ **Agnostic Bundling Integration**: Dynamic loading of the `"bun:sqlite"` driver inside the SQLite adapter to bypass Vitest's static analyzer and prevent built-in bundling errors while keeping `vitest.config.ts` database-agnostic.
- ✅ **Resilience Boot Validation**: Added test module unmocking in `integration-resilience.test.ts` to allow authentic configuration verification during startup and boot sequence validation, achieving 100% pass rates.
- ✅ **Clean Code Quality Gate**: Standardized catch blocks to parameterless catches to resolve all unused catch parameter lint errors.

### 2. Svelte 5 & UI Compilation ⚡

- ✅ **Store Effect Hardening**: Wrapped internal state variables in browser-safe checks (`!(globalThis as any).$effect`) in `src/stores/ui-store.svelte.ts`, eliminating `rune_missing_parentheses` compilation errors in modern Svelte 5 versions.

### 3. OpenAPI 3.1.0 Contract Verification 📖

- ✅ **Instant Generation**: Verified cached spec generation in **< 1 ms**, eliminating API discovery lag.
- ✅ **Dynamic Discovery**: Confirmed that new collections are automatically added to the spec.
- ✅ **Multilingual Mapping**: Verified translated fields as JSON Schema objects.

### 4. Dependency Injection (DI) Architecture 🏗️

- ✅ **SettingsService DI**: Successfully refactored `loadSettingsCache` to allow synthetic environment injection.
- ✅ **LocalCMS DI**: Hardened internal SDK testing by enabling `contentSystem` overrides.
- ✅ **Infrastructure Unblocking**: Resolved `bun-preload.ts` and `vite.config.ts` stubbing conflicts, allowing unit tests to run against real code while keeping client bundles slim.

### 5. Database-per-Worker Isolation ⚡

- ✅ **Parallel Speed**: E2E tests now run with `workers: 4` without SQLite locking issues.
- ✅ **Atomic State**: Each worker operates on a unique `.db` file, ensuring total data isolation.

### 6. Admin Theme System 🎨

- ✅ **Derived Values**: All AdminTheme density-based derived values verified (densityScale, spacingScale, sidebarWidth, headerHeight, stickyBarHeight, radii for card/input/button).
- ✅ **Visual Variant Logic**: cardShadow and cardBorder correctly synthesize per variant (flat → none, bordered → elevation, elevated → elevated shadow).
- ✅ **Feature Merge Strategy**: DEFAULT_THEME_FEATURES correctly merged with partial overrides; layoutRegions default to collections:left.
- ✅ **User Preference Overrides**: Tenant theme defaults preserved when user has no preference; user density/variant/features override tenant when set.
- ✅ **CSS Sanitization**: url(), @import, javascript:, <script>, and HTML tags stripped; safe CSS passes through unchanged.
- ✅ **SSR Component Propagation**: Compact and spacious densities correctly propagate CSS variables to Button, Card, Input, and Badge in server-rendered HTML.
- ✅ **Consolidation**: Replaced legacy `theme-context.test.ts` (5 tests) with 26-test suite in `admin-theme.test.ts`.

### 7. Enterprise Identity & SAML SSO / SCIM 2.0 🔐

- ✅ **SAML 2.0 CSRF State Hardening**: Modified authorization flow to generate dynamic, cryptographically secure 16-byte state tokens, storing them in a secure `saml_state` cookie (`SameSite: Lax`, `HttpOnly`) and validating them against the IdP's returned `RelayState` on the ACS endpoint to prevent OAuth/SAML CSRF.
- ✅ **SCIM 2.0 Bulk Endpoint**: Implemented support for the SCIM `POST /Bulk` endpoint, handling multi-operation payloads, `failOnErrors` constraints, and resolving nested `bulkId:<id>` references dynamically to link newly created users with group/role associations in a single request.

## 2. Health by layer

| Layer                       | Confidence         | What it proves                                                                 | How to run                                                        |
| --------------------------- | ------------------ | ------------------------------------------------------------------------------ | ----------------------------------------------------------------- |
| **Unit** (Vitest)           | **High**           | Auth cookies, import-boundary, page gates, widgets, security, dispatcher edges | `bun run test:unit`                                               |
| **Security regression**     | **High**           | Hooks + RBAC defense-in-depth (~2s local)                                      | See AGENTS.md security command                                    |
| **Integration** (black-box) | **High on SQLite** | HTTP + DB; **session→`__data` page loads**; multi-adapter matrix in CI         | `bun run scripts/run-integration-tests.ts --db=sqlite --no-build` |
| **E2E** (Playwright)        | **Medium–High**    | Browser goldens / P0 journeys; not API matrices                                | `bun x playwright test --project=chromium`                        |
| **Benchmarks**              | **Selective**      | Perf matrix; full suite is long — not every PR                                 | See [benchmarks](../project/benchmarks/index.mdx)                 |

**Gates (local → CI)**

| When       | What runs (summary)                                                  |
| ---------- | -------------------------------------------------------------------- |
| Pre-commit | Format/lint/slop on staged + fast security set                       |
| Pre-push   | Full unit, build (when needed), optional DB tasks, P0 coverage check |
| CI         | Matrix: lint/types/unit → build → integration × DBs → E2E projects   |

**CI smart reporting** (job Summary tab + annotations):

| Job                   | Reporter                                                                                         |
| --------------------- | ------------------------------------------------------------------------------------------------ |
| E2E prep / E2E shards | `scripts/ci-report-playwright.ts` — pass/fail counts, first error, fix hints, `preview.log` tail |
| Integration per DB    | runner writes table + failure stderr; `db-summary` aggregates adapters                           |
| All Green             | `scripts/ci-report-all-green.ts` — job matrix + fix order                                        |

## 5. What to run locally

```bash
# Daily / before commit
bun run gate:fast

# Before push (matches pre-push spirit)
bun run verify:push

# Unit only
bun run test:unit

# Ultra-smart unit slice (auth, boundary, queue/redirects remotes)
bun run test:unit -- tests/unit/auth/session-cookies.test.ts tests/unit/live/client-import-boundary.test.ts tests/unit/api/testing-login-cookie-contract.test.ts tests/unit/utils/page-guards.test.ts tests/unit/routes/collectionbuilder-page-server.test.ts tests/unit/routes/queue-remote-admin.test.ts tests/unit/routes/queue-actions.server.test.ts tests/unit/routes/redirects-remote-admin.test.ts tests/unit/api/namespace-ownership.test.ts

# Integration (SQLite) — session-page-load, webhooks, automations, config surface, security-negative
bun run scripts/run-integration-tests.ts --db=sqlite --no-build

# E2E (example: user journeys)
bun x playwright test tests/e2e/routes/user/p0-journeys.spec.ts

# Full local CI-ish (incl. E2E — slow)
bun run ci:local
```

Safety: tests must never point at production DBs — see test-config safety guards in [git-workflow.mdx](./git-workflow.mdx).

## 7. How to keep this page honest

When you ship a meaningful test or stability change:

1. Update **this file** only if health, open gaps, or “hot areas” change (few sentences + table rows).
2. Put deep numbers, file inventories, and “resolved” write-ups in the **[archive](./test-status-archive.mdx)** (or route-specific docs).
3. Never treat static totals as CI truth — link to Actions.

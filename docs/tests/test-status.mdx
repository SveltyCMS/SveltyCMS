**Hardening applied**:

- Security rate-limiter reset between tests ŌåÆ zero cross-test leakage
- `IDLE` state accepted in health checks ŌåÆ eliminates 10s settle delay
- Widget auto-registration ŌåÆ fixes 404 for valid widgets in fresh DB
- All caches + state stores cleared per-test ŌåÆ true isolation
- **Test DB safety guard** (`scripts/check-test-db-safety.ts`) runs in pre-commit and pre-push hooks ŌåÆ blocks unsafe `config/private.test.ts` before tests start
- **Shared classifier** (`src/utils/test-db-safety.ts`) used by server, integration runner, and git hooks ŌåÆ single source of truth for isolated DB name validation
- **Secret synchronization** ŌĆö integration runner writes `TEST_API_SECRET` to `tests/e2e/.auth/test-secret.txt` before server start ŌåÆ eliminates 401 mismatch
- **`SKIP_COLLECTION_COMPILE`** flag prevents @src alias resolution failures during integration test builds on Windows

These measures ensure zero cross-test contamination and deterministic test results.

### Running Tests Locally

```bash
# Run the High-Performance Quality Gate (4.4x faster)
bun run scripts/quality-gate.ts

# Run the complete Enterprise Performance Matrix (All 4 Databases)
bun run scripts/benchmark-matrix/index.ts

# Run a specific benchmark (Smart-Redirect Enabled)
# You can use 'bun run', but the system will automatically redirect to 'bun test'
# for maximum precision and automatic resource cleanup.
bun test tests/benchmarks/api-latency.test.ts
```

> [!TIP]
> SveltyCMS benchmarks are **Environment-Aware**. If you execute a benchmark file with `bun run`, the system will automatically inform you and reroute the execution through the **Bun Test Engine** to ensure accurate micro-latency measurements and proper cleanup of background services.

## Enterprise TQA (Total Quality Assurance) Methodology

SveltyCMS has adopted a **Three-Pillar Total Quality Assurance (TQA)** model to guarantee enterprise-grade resilience, going beyond standard functional verification. This methodology is the standard for all development efforts.

### 1. White-Box Testing (Unit)

**"Test the code knowing how it works."**

- **Focus:** Logic correctness, edge cases, internal consistency.
- **Tools:** Vitest / Bun Unit Tests.
- **What we test:** Pure functions, utility validators, service methods, and Svelte 5 reactivity logic.
- **Benefit:** Provides high speed and deep coverage, ensuring internal logic is mathematically sound.

### 2. Black-Box Testing (Integration & E2E)

**"Test the system as a consumer sees it."**

- **Focus:** API Contracts, Security Boundaries, Real-World Behavior.
- **Tools:** Playwright (E2E) & API Integration Tests.
- **What we test:** The entire middleware chain (Firewall $\to$ Auth $\to$ Logic $\to$ DB). The system is treated as an opaque box, guaranteeing **Database Agnosticism** and **Fail-Closed Authorization**.

### 3. Audit (High-Frequency Benchmark)

**"Reality simulation with extreme stress and chaos resilience."**

- **Focus:** Performance and System Stability (SRE/Chaos Engineering).
- **Tools:** Benchmark Matrix (`index.ts`).
- **What we test:**
  - **Concurrency:** Lost Update Race Conditions (Atomic Consistency).
  - **Temporal Integrity:** Timezone normalization and data contracts.
  - **Chaos:** System stability during simulated DB brownouts.
  - **Load:** Stress-testing 100,000+ collections (Index Pressure).

SveltyCMS uses a strict isolation strategy to ensure tests never touch production data.

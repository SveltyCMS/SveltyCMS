### Running Tests Locally

```bash
# Run all unit tests (Vitest/jsdom)
bun run test:unit

# Run the complete Enterprise Performance Matrix (All 4 Databases)
# Orchestrator core lives in scripts/benchmark-matrix/
bun run scripts/benchmark-matrix/index.ts

# Run an individual autonomous benchmark (Decentralized Pattern)
# Each test now manages its own ephemeral server instance.
bun test tests/benchmarks/api-latency.test.ts
```

## Test Architecture & Isolation

SveltyCMS employs a strict isolation strategy to ensure tests never touch production data. Standardization on **127.0.0.1** with **dynamic port assignment** (via PID-based logic) for benchmarks ensures that concurrent test runs never conflict.

## Test Frameworks Overview

- **Purpose**: Fast validation of business logic, services, and API endpoints.
- **Location**: `tests/unit/`
- **Test Count**: 670 tests (consolidated for performance)
- **Execution Time**: ~8 seconds

**What It Tests**:

- ✅ **Middleware Hooks**: 71% latency reduction and fast-path short-circuits.
- ✅ **Database System**: Unified adapter logic with micro-telemetry verification.
- ✅ **Dependency Injection (DI)**: Deterministic isolation for core services (`SettingsService`, `LocalCMS`).
- ✅ **Secure Preview**: Signed HMAC-SHA256 handshake tokens and expiration logic.
- ✅ **Audit Logging**: Non-blocking mutation interceptor with duration tracking.
- ✅ **Fail-Closed API Security**: Granular authorization gatekeeper for the dispatcher.
- ✅ **OpenAPI 3.1.0 Contract**: Dynamic schema discovery and JSON Schema integrity.
- ✅ **Environment Parity**: jsdom (Vitest) for component logic + Native (Bun) for core utilities.

### 🚀 Performance Benchmarks

- **Audit 2.0 Matrix**: High-fidelity cross-database benchmarking with intensity locking.
- **Hooks Benchmark**: Measures micro-latency of the middleware chain (target < 0.6ms).
- **Database Benchmark**: Measures CRUD micro-latency and telemetry integrity.
- **Media Benchmark**: Measures upload block-time reduction via async job queue (target 99%+ reduction).

### 🎭 Playwright - End-to-End Tests

- **Purpose**: Browser automation for critical user journeys.
- **Location**: `tests/e2e/`
- **Test Count**: ~40 tests

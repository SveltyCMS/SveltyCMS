# Getting Started with SveltyCMS

Get up and running in minutes — whether you're a **Developer** installing the system, an **Administrator** configuring permissions, or a **Content Creator** building your first site.

> 💡 **Default**: SQLite (zero-config, in-process, sub-millisecond). No Docker, no external database needed.
> Upgrade to PostgreSQL/MongoDB/MariaDB when you need multi-instance scale.

## 1. Installation (For Developers)

SveltyCMS features a **Self-Healing Setup Wizard** that automates database configuration and environment setup.

### Quick Install

#### Option A: Scaffolding CLI (recommended)

```bash
npx create-sveltycms@latest my-cms
cd my-cms
bun run dev
```

Templates: `website`, `blog`, `agency`, `saas`, `corporate`, `ecommerce` · Databases: `sqlite` (default), `postgresql`, `mariadb`, `mongodb`

#### Option B: Clone from GitHub

```bash
git clone https://github.com/SveltyCMS/SveltyCMS.git
cd SveltyCMS
bun install
bun run dev
```

Navigate to `http://localhost:5173`. The system detects the absence of `config/private.ts` and launches the self-healing Setup Wizard.

| Step                    | What Happens                                                                                                                |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| 1. **Choose Database**  | **SQLite** (default, fastest for local dev), PostgreSQL, MariaDB, or MongoDB                                                |
| 2. **Test Connection**  | Auto-installs required drivers, validates connectivity                                                                      |
| 3. **Create Admin**     | Set master credentials with Argon2id password hashing                                                                       |
| 4. **Configure System** | Site name, languages, media storage, optional Redis                                                                         |
| 5. **Email & Complete** | Optionally configure SMTP (optional), review settings, complete setup. System transitions to READY with progressive warmup. |

The system seeds the database and transitions to `READY` — no manual config file editing.

> 💡 After setup, all `/api/setup` endpoints are permanently blocked (403) as a security measure.

## 2. First Steps (For Admins & Editors)

### Access the CMS

Once setup completes, you will be automatically redirected to the CMS dashboard. If you need to log in again, navigate to `http://localhost:5173/login` and use your admin credentials.

### Understanding the CMS Layout

The CMS is designed for clarity and speed.

```text
┌─────────────────────────────────────────────┐
│ 🏠 SveltyCMS            [Search] 👤 Admin ▼ │
├──────────┬──────────────────────────────────┤
│          │                                   │
│ 🏠 Home  │   📊 Dashboard Overview           │
│ 📚 Collections                              │
│ 📁 Media │   • System Health                │
│ 👥 Users │   • Recent Content               │
│ ⚙️ Config    • Online Users                │
│          │   • Storage Usage                │
│          │                                   │
└──────────┴──────────────────────────────────┘
```

### First Tasks

1.  **Configure Settings**: Go to **⚙️ Config** to set your Site Name, Default Language, and Theme.
2.  **View your homepage**: With the default **Website Starter** preset, visit `http://localhost:5173/` — guests see a published homepage designed with [Svedit](/docs/development/site-starter.mdx).
3.  **Design the frontpage**: Edit **Pages → Home** in the form (free), or use **Live Preview** for inline Svedit editing ([Editable Website plugin](/src/plugins/editable-website/editable-website.mdx) — 14-day trial, auto-enabled on Website Starter).
4.  **Add more content**: Create additional `pages` entries (`about`, `contact`, …) or add collections from templates (Blog, Agency, SaaS, Corporate, E-commerce).

## 3. Common Workflow Templates

### Website Starter (recommended — SvelteKit + Svedit)

**Website Starter** is pre-selected in the Setup Wizard. After setup you get:

1. In-repo **SvelteKit** public routes at `routes/(site)`
2. A **Pages** collection with a published `home` entry (Svedit layout in `content`)
3. **Svedit** inline block editing when Live Preview is enabled
4. **Live Preview bridge** (trial included): [Editable Website](/src/plugins/editable-website/editable-website.mdx) (€14.99 after trial) — iframe sync and inline Svedit editing; CMS form editing stays free

See the [Site Starter guide](/docs/development/site-starter.mdx). For Astro/Next.js/Vue, disable the site starter and consume the headless API instead.

### Blog

1. Create collections: `Posts`, `Categories`, `Tags`
2. Link `Posts` → `Categories` (multi-select) and `Users` (author)
3. Use the `Status` field for Draft → Published workflow
4. Enable comments or SEO metadata via widget options

### E-Commerce Catalog

1. Create collections: `Products` (Price, SKU, Description), `Inventory`
2. Upload images to 📁 Media Hub, link to entries
3. Query via `/api/collections/products` for your frontend
4. Add Stripe plugin for payment processing

### Multi-Language

1.  **Enable i18n**: Go to **⚙️ Config** → **Languages** and add your target languages.
2.  **Translate**: Open any entry and use the language switcher to enter localized content. Paraglide JS handles the compile-time switching.

## 🛡️ User Roles & Permissions

Granular **RBAC** with field-level access control:

- **Admin**: Full access.
- **Editor**: Can manage all content and media.
- **Contributor**: Can create content but cannot publish without approval.
- **Custom Roles**: Create these in **⚙️ Config** → **Roles & Permissions** to define field-level access.

# Getting Started with SveltyCMS

Welcome to SveltyCMS! This guide will help you get up and running in under 10 minutes, whether you are a **Developer** installing the system, an **Administrator** configuring permissions, or a **Content Creator** building your first site.

## 1. Installation (For Developers)

SveltyCMS features a **Self-Healing Setup Wizard** that automates database configuration and environment setup.

### Quick Install

```bash
# 1. Clone the repository
git clone https://github.com/SveltyCMS/SveltyCMS.git
cd SveltyCMS

# 2. Install dependencies (Bun recommended)
bun install
#   Note for Windows users: If bun install fails, use npm install instead

# 3. Start development server
bun run dev
```

### The Setup Wizard

Navigate to `http://localhost:5173`. The system will automatically detect the absence of `config/private.ts` via the TurboPipeline hook and launch the wizard:

1.  **Choose Database**: Support for MongoDB, PostgreSQL, MariaDB, and SQLite (fastest for local).
2.  **Test Connection**: The wizard auto-installs required drivers and validates connectivity.
3.  **Create Admin**: Set your master credentials with password complexity validation.
4.  **Configure System**: Set site name, languages, media storage, and optional Redis.
5.  **Email & Review**: Optionally configure SMTP, then review all settings and complete setup. The system seeds the database automatically and transitions to `READY`.

## 2. First Steps (For Admins & Editors)

### Access the CMS

Once setup completes, you will be automatically redirected to the CMS dashboard. If you need to log in again, navigate to `http://localhost:5173/login` and use your admin credentials.

### Understanding the CMS Layout

The CMS is designed for clarity and speed.

```
┌─────────────────────────────────────────────┐
│ 🏠 SveltyCMS            [Search] 👤 Admin ▼ │
├──────────┬──────────────────────────────────┤
│          │                                   │
│ 🏠 Home  │   📊 Dashboard Overview           │
│ 📚 Collections                              │
│ 📁 Media │   • System Health                │
│ 👥 Users │   • Recent Content               │
│ ⚙️ Config    • Online Users                │
│          │   • Storage Usage                │
│          │                                   │
└──────────┴──────────────────────────────────┘
```

### Key Editorial Tasks

1.  **Configure Settings**: Go to **⚙️ Config** to set your Site Name, Default Language, and Theme.
2.  **Create a Collection**: Collections define your content structure. Go to **📚 Collections** → **+ New Collection**.
    - _Example_: Create a "posts" collection with fields for Title (text), Content (richtext), and Status (select).
3.  **Add Your First Entry**: Open your new collection and click **+ New Entry**. Fill in the fields and click **Save**.

## 3. Common Workflow Templates

### Workflow 1: Blog Setup

1.  **Collections**: Create `Posts`, `Categories`, and `Tags`.
2.  **Relationships**: Link `Posts` to `Categories` (Multi-select) and `Users` (Author).
3.  **Publishing**: Use the built-in `Status` field to manage Draft vs. Published states.

### Workflow 2: E-Commerce Catalog

1.  **Collections**: Create `Products` (Price, SKU, Description) and `Inventory`.
2.  **Media**: Upload product images to the **📁 Media Hub** and link them to product entries.
3.  **API**: Query products via `/api/collections/products` for your frontend.

### Workflow 3: Multi-Language Sites

1.  **Enable i18n**: Go to **⚙️ Config** → **Languages** and add your target languages.
2.  **Translate**: Open any entry and use the language switcher to enter localized content. Paraglide JS handles the compile-time switching.

## 🛡️ User Roles & Permissions

SveltyCMS uses a granular **Role-Based Access Control (RBAC)** system.

- **Admin**: Full access.
- **Editor**: Can manage all content and media.
- **Contributor**: Can create content but cannot publish without approval.
- **Custom Roles**: Create these in **⚙️ Config** → **Roles & Permissions** to define field-level access.

## ⚙️ How it Works (The Mechanics)

For developers needing to understand the system's resilience:

### The State Machine

SveltyCMS transitions through specific states during boot:

- `IDLE`: Initializing memory and checking `config/private.ts`.
- `READY` / `WARMED`: Core services (Auth, Media, DB) are successfully connected and cache is warm.
- `DEGRADED`: System is operational but some non-critical services are unhealthy.
- `FAILED`: A critical path error occurred (Wizard enters recovery mode).

### Driver Auto-Installation

During the Setup Wizard, SveltyCMS uses `spawn` to dynamically install database drivers (e.g., `mongoose`, `mariadb`) based on your selection, ensuring a lean initial footprint.

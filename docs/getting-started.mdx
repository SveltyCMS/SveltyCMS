## 1. Installation (For Developers)

SveltyCMS features a **Self-Healing Setup Wizard** that automates database configuration and environment setup.

### Quick Install

```bash
# 1. Clone the repository
git clone https://github.com/SveltyCMS/SveltyCMS.git
cd SveltyCMS

# 2. Install dependencies (Bun recommended)
bun install
# Windows users: if bun install fails, use `npm install` instead

# 3. Start development server
bun run dev
```

### Setup Wizard Flow

Navigate to `http://localhost:5173`. The system detects the absence of `config/private.ts` and launches the wizard:

| Step                    | What Happens                                                    |
| ----------------------- | --------------------------------------------------------------- |
| 1. **Choose Database**  | MongoDB, PostgreSQL, MariaDB, or SQLite (fastest for local dev) |
| 2. **Test Connection**  | Auto-installs required drivers, validates connectivity          |
| 3. **Create Admin**     | Set master credentials with Argon2id password hashing           |
| 4. **Configure System** | Site name, languages, media storage, optional Redis             |
| 5. **Email & Review**   | Optionally configure SMTP, review all settings, complete setup  |

The system seeds the database and transitions to `READY` — no manual config file editing.

> 💡 After setup, all `/api/setup` endpoints are permanently blocked (403) as a security measure.

## 2. First Steps (For Admins & Editors)

### Access the CMS

After setup, you're redirected to the dashboard. To log back in: `http://localhost:5173/login`.

### CMS Layout

```text
┌─────────────────────────────────────────────┐
│ 🏠 SveltyCMS            [Search] 👤 Admin ▼ │
├──────────┬──────────────────────────────────┤
│          │                                   │
│ 🏠 Home  │   📊 Dashboard                    │
│ 📚 Coll. │   • System Health                 │
│ 📁 Media │   • Recent Content                │
│ 👥 Users │   • Online Users                  │
│ ⚙️ Config│   • Storage Usage                 │
│          │                                   │
└──────────┴──────────────────────────────────┘
```

### First Tasks

1. **Configure Settings** → ⚙️ Config: Site Name, Default Language, Theme
2. **Create a Collection** → 📚 Collections → **+ New Collection**
   - _Example_: "Posts" with Title (Text), Content (RichText), Status (Select)
3. **Add an Entry** → Open your collection → **+ New Entry** → Fill fields → **Save**

## 3. Common Workflow Templates

### Blog

1. Create collections: `Posts`, `Categories`, `Tags`
2. Link `Posts` → `Categories` (multi-select) and `Users` (author)
3. Use the `Status` field for Draft → Published workflow
4. Enable comments or SEO metadata via widget options

### E-Commerce Catalog

1. Create collections: `Products` (Price, SKU, Description), `Inventory`
2. Upload images to 📁 Media Hub, link to entries
3. Query via `/api/collections/products` for your frontend
4. Add Stripe plugin for payment processing

### Multi-Language

1. Go to ⚙️ Config → Languages → add target languages
2. Open any entry → use the per-field language switcher
3. Paraglide JS compiles translations at build time — zero runtime overhead

## 🛡️ User Roles & Permissions

Granular **RBAC** with field-level access control:

| Role            | Access                                                            |
| --------------- | ----------------------------------------------------------------- |
| **Admin**       | Full system access                                                |
| **Editor**      | Manage all content and media                                      |
| **Contributor** | Create content, cannot publish without approval                   |
| **Custom**      | Define per-collection, per-field permissions in ⚙️ Config → Roles |

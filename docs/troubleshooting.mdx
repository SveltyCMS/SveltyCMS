# Troubleshooting Guide

This guide provides solutions for common issues encountered during installation, configuration, and daily operation.

---

## 🚀 Setup & Installation Issues

### Goal: Fix "Initializing System" Hang

**The Problem**: The splash screen stays on "Initializing System" indefinitely or throws a MIME type error (`text/html` instead of `javascript`).

**🛠️ The Solution:**

1.  **Check `config/private.ts`**: If the file is empty or corrupted, the Setup Wizard cannot move to `READY`. Delete the file and restart `bun run dev`.
2.  **Verify Vite Cache**: Vite's dependency optimization can sometimes desync.
    ```bash
    rm -rf node_modules/.vite
    bun run dev
    ```
3.  **Check Middleware Logic**: Ensure `hooks.server.ts` is not crashing before the `handleSystemState` stage.

---

### Goal: Fix Database Connection Failures

**The Problem**: Error message: `Check your credentials` or `ECONNREFUSED`.

**🛠️ The Solution:**

- **MongoDB**: Ensure your cluster hostname is correct and your IP is whitelisted in Atlas.
- **PostgreSQL/MariaDB**: Ensure the database exists _before_ running the wizard. SveltyCMS scopes by `database_name`.
- **SQLite**: Ensure the directory for the `.db` file has write permissions.

**⚙️ Mechanics:**
Check `src/databases/db.ts` to see how the adapter probes the connection. Use `bun run check:mongodb` (if available) for isolated testing.

## 🔐 Authentication & Session Issues

### Goal: Resolve "Unauthorized" or "Session Expired"

**The Problem**: Getting logged out immediately or seeing `401 Unauthorized` on all API calls.

**🛠️ The Solution:**

1.  **Clear Browser Storage**: Sessions are tied to cookies. Clear cookies for your domain and try again.
2.  **Verify `JWT_SECRET`**: If you manually edited `config/private.ts` and changed the secret, all active sessions are invalidated.
3.  **Check Node.js Time**: Ensure your server's system time is synchronized. Divergent times can cause JWT expiration errors.

## 🧩 Widget & Collection Issues

### Goal: Fix Missing Widgets in Dashboard

**The Problem**: You added a widget to `src/widgets/core/`, but it doesn't appear in the Studio Dashboard.

**🛠️ The Solution:**

- Use the **"Sync Widgets"** button in the `System Settings` → `Widgets` panel.
- **Mechanics**: The SveltyCMS scanner (`collectionScanner.ts`) runs on HMR, but the database registry needs a manual trigger to update for new custom widgets.

## 📈 Performance & Memory

### Goal: Fix "JavaScript heap out of memory"

**The Problem**: The build process fails on large media libraries or complex collections.

**🛠️ The Solution:**
Increase the Node.js memory limit:

```bash
# Set limit to 4GB
export NODE_OPTIONS="--max-old-space-size=4096"
bun run build
```

## 💻 Environment & OS-Specific Issues

### Goal: Fix Windows `bun install` Corruption

**The Problem**: `bun install` version 1.3.11+ on Windows may result in corrupted `node_modules`. This is an upstream bug where the installer writes null bytes into `package.json` files and leaves package directories empty.

**Symptoms**:

- `bun install` fails with a `ParserError` for packages like `@sveltejs/kit` or `esbuild`.
- After a seemingly successful install, `bun run dev` fails with `Error: Invalid package config`.
- Checking `node_modules` shows many `package.json` files containing only null bytes or empty directories.

**🛠️ The Solution**: Use `npm install` instead of `bun install`.

```bash
npm install    # Works reliably on Windows
bun run dev    # bun's runtime/runner works fine once packages are installed
```

_Note: This only affects the Bun installer on Windows; Bun's runtime and other commands remain fully functional._

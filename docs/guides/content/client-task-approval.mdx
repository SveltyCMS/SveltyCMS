# Client Task Approval System

**A complete solution for managing client work requests with approval workflows and billing.**

Perfect for agencies, freelancers, and development teams who need client approval before starting work.

---

## Quick Overview

The Client Task Approval System provides three interconnected collections:

1. **Clients** - Store client information and hourly rates
2. **Tasks** - Manage work requests with approval workflows  
3. **Invoices** - Generate bills for completed work

## Installation

### Quick Install

Copy the example collections to your SveltyCMS instance:

```bash
cp docs/examples/collections/*.ts config/collections/Collections/
bun run dev
```

The collections will appear in your admin panel under Collections.

### Manual Setup

Alternatively, create the collections through the Collection Builder GUI using the example files as templates.

## Basic Workflow

### 1. Add a Client

Navigate to **Collections ‚Üí Clients ‚Üí Create New Entry**

```

Email: contact@acme.com
Hourly Rate: $150.00
```

### 2. Create a Task

Navigate to **Collections ‚Üí Tasks ‚Üí Create New Entry**

```

Client: [Select Acme Corp]
Estimated Hours: 12
Estimated Cost: $1,800 (12 √ó $150)
Approval Status: Pending
Work Status: Not Started
```

### 3. Get Client Approval

Share task details with client. When they approve:

```

Approval Date: 2024-01-15
```

### 4. Do the Work

Update as you progress:

```

Actual Hours: 10.5
Completion Date: 2024-01-20
```

### 5. Generate Invoice

Navigate to **Collections ‚Üí Invoices ‚Üí Create New Entry**

```

```

### 6. Track Payment

When payment is received:

```

```

## API Examples

### REST API

```bash
# Get pending tasks
GET /api/collections/tasks?filter[approvalStatus]=pending

# Get unpaid invoices
GET /api/collections/invoices?filter[paymentStatus]=sent
```

### GraphQL

```graphql

query {
  tasks(filter: { approvalStatus: "pending" }) {
    id
    taskTitle
    estimatedCost
    client {
      clientName
      hourlyRate
    }
  }
}
```

## Troubleshooting

**Q: Can I change a client's hourly rate?**  
A: Yes, but it only affects new tasks. Existing tasks keep their original cost.

**Q: What if actual hours exceed estimate?**  
A: Bill for actual hours. Update the cost in the invoice line items accordingly.

**Q: How do I handle discounts?**  
A: Add a line item in the invoice with a negative amount, or adjust the total.

**Q: Can I use different currencies per client?**  
A: Yes, edit each client's hourly rate field and change the `currencyCode`.

## Security Notes

- **Field Permissions**: Configure who can view/edit hourly rates and invoices
- **Internal Notes**: Consider hiding internal notes from client-facing views
- **Tenant Isolation**: Multi-tenant support ensures clients only see their data
- **Audit Trail**: Revision history tracks all changes for accountability

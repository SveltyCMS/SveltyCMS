# Widget System Architecture

This document details the internal mechanisms of the SveltyCMS Widget System, specifically the **3-Pillar Architecture**.

## 🏗️ The 3 Pillars

1.  **Definition Pillar (`index.ts`)**: The immutable blueprint.
2.  **Input Pillar (`input.svelte`)**: The interactive editor.
3.  **Display Pillar (`display.svelte`)**: The read-only view.

## 🏭 Factory System

Standardized widget creation via `createWidget()` ensures type safety and consistent data contracts.

## 🧩 Discovery System

- **Core**: Built-in widgets.
- **Custom**: Portable modules in `src/widgets/custom/`.
- **Marketplace**: Dynamically loaded extensions.

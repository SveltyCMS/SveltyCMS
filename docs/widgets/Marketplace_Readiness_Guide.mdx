## ğŸ“„ 1. The Manifest (widget.json)

Your `widget.json` is the source of truth for the Marketplace. It MUST be valid JSON.

### Required Fields

```json
{
	"name": "MyCoolWidget", // Unique PascalCase identifier
	"version": "1.0.0", // SemVer version
	"description": "Short description for listing",
	"author": "Your Name <you@example.com>",
	"license": "MIT",
	"icon": "mdi:rocket", // Iconify ID
	"category": "custom", // Core category
	"tags": ["cool", "feature"], // Search tags
	"compatibility": {
		"sveltycms": "^2.0.0" // Supported CMS versions
	}
}
```

### Permission Declarations

You must declare all permissions your widget requires. **Undocumented permissions will cause rejection.**

```json
  "permissions": [
    "collection:read",
    "media:upload"
  ]
```

## ğŸ”’ 2. Security Standards

The Marketplace enforces strict security rules. Your code will be scanned upon submission.

### ğŸš« Forbidden Patterns

- **`eval()` / `new Function()`**: Remote code execution risk.
- **`innerHTML` without sanitization**: XSS risk. Use the system `Sanitize` component.
- **Hardcoded Secrets**: API keys or credentials committed to code.
- **External scripts**: Loading scripts from CDNs without integrity hashes.

### âœ… Allowed Practices

- Use SveltyCMS internal APIs for database access.
- Use `fetch` for external API calls, but document the endpoints.
- Sanitize all user inputs using standard libraries.

## ğŸ“ 4. Documentation

A `README.md` is mandatory. It should include:

1.  **Installation**: Simple steps (if manual).
2.  **Configuration**: Explanation of options in `widget.json`.
3.  **Usage**: Screenshots or code snippets showing the widget in action.
4.  **Support**: Where to report bugs or get help.

## â“ Frequently Asked Questions

**Q: Can I update my widget after publishing?**
A: Yes, increment the version in `widget.json` and resubmit.

**Q: Are paid widgets allowed?**
A: Yes, the marketplace supports commercial licensing.

**Q: Can I use 3rd party npm packages?**
A: Yes, but keep dependencies minimal. Peer dependencies on SveltyCMS core are preferred.

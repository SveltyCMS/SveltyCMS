## 3. The Mechanics

### Documentation Integrity
The API documentation is integrated into the CI/CD pipeline. Every PR that modifies a file in `src/routes/api/[...path]/handlers/` must include a corresponding update to its `.mdx` reference in `docs/api/`.

### Verification Strategy
- **Linter**: `bun ./tests/lint-docs.ts` verifies that all URL segments mentioned in docs exist as logic in the handlers.
- **SDK Parity**: Ensures that every REST endpoint documented has a corresponding `locals.cms` method for SSR.

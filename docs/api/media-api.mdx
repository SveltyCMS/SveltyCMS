# Media API Reference

The SveltyCMS Media API provides a robust, type-safe system for managing digital assets. It features a **3-Tier Processing Pipeline**: Secure Ingestion, AI Enrichment, and Dynamic Delivery.

## 🚀 Quick Reference

| Method     | Endpoint             | **Local SDK Equivalent**    | Description                           |
| :--------- | :------------------- | :-------------------------- | :------------------------------------ |
| **POST**   | `/api/media/process` | `locals.cms.media.upload`   | Upload files (multipart/form-data)    |
| **GET**    | `/api/media`         | `locals.cms.media.find`     | List media files (paginated)          |
| **GET**    | `/api/media/[id]`    | `locals.cms.media.findById` | Get media details                     |
| **PATCH**  | `/api/media/[id]`    | `locals.cms.media.update`   | Update metadata (alt, tags, filename) |
| **DELETE** | `/api/media/[id]`    | `locals.cms.media.delete`   | Delete media permanently              |

> **Use the Local SDK**: For server-side tasks like processing a form action or generating an image gallery in `+page.server.ts`, **always use `locals.cms.media`**. It is **10-20x faster** than using `fetch()` because it communicates directly with the database and file storage.

## 1. MediaItem Data Structure

Every asset in SveltyCMS is represented by a `MediaItem` object.

```typescript
interface MediaItem {
  _id: string; // Unique Database ID
  filename: string; // Current sanitized filename
  originalFilename: string; // Filename at time of upload
  hash: string; // SHA-256 content hash (prevents duplicates)
  path: string; // Relative storage path
  mimeType: string; // e.g., "image/webp", "video/mp4"
  size: number; // Size in bytes
  access: "public" | "private" | "protected";
  createdBy: string; // User ID of uploader
  createdAt: string; // ISO Timestamp
  updatedAt: string; // ISO Timestamp
  folderId?: string; // Virtual folder association
  thumbnails: Record<string, { url: string; width: number; height: number }>;
  metadata: {
    width?: number;
    height?: number;
    duration?: number;
    aiTags?: string[]; // Generated by Ollama
    focalPoint?: { x: number; y: number };
    exif?: Record<string, unknown>;
    iptc?: Record<string, unknown>;
    xmp?: Record<string, unknown>;
    author?: string;
    copyright?: string;
    [key: string]: any; // Extensible for DAM data
  };
}
```

## 2. Secure Ingestion

### A. Local Upload (POST)

**Endpoint**: `/api/media/process`
**Payload**: `multipart/form-data` with one or more `files`.

### B. Remote Ingestion (POST)

**Endpoint**: `/api/media/remote`
**Payload**: `{"url": "https://..."}`

> [!IMPORTANT]
> **SSRF Protection**: SveltyCMS uses a hardened gateway that blocks all non-public IP ranges (RFC 1918) and loopback addresses.

### C. Bulk Download

To download multiple files, use the bulk-download endpoint.

```javascript
// Client-side implementation
const response = await fetch("/api/media/bulk-download", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ fileIds: ["id1", "id2"] }),
});
const blob = await response.blob();
const url = URL.createObjectURL(blob);
const a = document.createElement("a");
a.href = url;
a.download = "archive.tar.gz";
a.click();
```

## 3. AI Intelligence (optional)

If `USE_AI_TAGGING` is enabled in System Settings, SveltyCMS can leverage local LLMs (via Ollama) for asset enrichment.

### A. Auto-Tagging

**Endpoint**: `POST /api/media/ai-tag`
**Action**: Sends the image buffer to a vision-capable model (e.g., Llava) to generate descriptive tags.

### B. Smart Focal Points

**Endpoint**: `PATCH /api/media/[id]/focal`
**Logic**: Sets the "center of interest" for the dynamic transformation engine.

## 4. Advanced Search

The Search API allows for granular asset discovery using the following criteria:

| Criteria                | Description                                                |
| :---------------------- | :--------------------------------------------------------- |
| `filename`              | Partial match on the filename.                             |
| `tags`                  | Array of tags. Returns items matching ANY tag.             |
| `minWidth` / `maxWidth` | Filter images by pixel dimensions.                         |
| `aspectRatio`           | `landscape`, `portrait`, or `square`.                      |
| `minSize` / `maxSize`   | Filter by file weight in bytes.                            |
| `fileTypes`             | Array of MIME types (e.g., `["image/jpeg", "image/png"]`). |
| `hasEXIF`               | Boolean. Filters images with embedded metadata.            |
| `dominantColor`         | Hex string (e.g., `#FF5733`).                              |
| `showDuplicatesOnly`    | Returns items sharing the same `hash`.                     |

## 5. Security & Hardening

SveltyCMS treats the Media Hub as a high-risk security vector. The following protections are mandatory:

- **Path Resolution**: All filesystem requests are strictly sandboxed. Any path containing `..` or leading outside the `MEDIA_ROOT` is rejected with `403 FORBIDDEN`.
- **MIME Validation**: We do not trust the `Content-Type` header from the client. Every upload is inspected via our native **SlimSniffer** utility to determine the true file signature using Magic Bytes, without heavy third-party dependencies.
- **Command Injection**: Image processing (via Sharp) and Video transcoding (via FFmpeg) are executed in isolated processes with strict argument sanitization.

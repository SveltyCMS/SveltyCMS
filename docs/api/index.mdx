## 🛡️ Unified Gatekeeper Architecture

As of April 2026, SveltyCMS has consolidated its REST surface from fragmented per-resource routes to a high-performance **Unified Gatekeeper** (`src/routes/api/[...path]`).

### Key Benefits

- **Zero-Latency Dispatching**: Internal server-to-server calls bypass HTTP overhead via the `LocalCMS` bridge.
- **Fail-Closed Security**: Every request must pass the central `CheckAuthorization` before being dispatched to domain handlers.
- **Simplified Deployment**: Reduced bundle size and faster warm-ups (2.0s → 0.8s) by removing redundant route definitions.
- **Cross-Domain Consistency**: All services (Auth, Content, Media, System) share the same validation and error-handling middleware.

---

## 🚀 Why the API is "Enterprise & Headless Ready"

1.  **Zero-Latency Local SDK (The "Internal" Tier)**: In a standard headless setup, a page load might require 3–5 API calls, each taking 50–100ms. SveltyCMS uses the `locals.cms` SDK to interact directly with the database memory space, achieving **0.1ms – 5ms** overhead.
2.  **JIT-Compiled GraphQL (The "Intelligent" Tier)**: For external consumers, we use `@envelop/graphql-jit`. Instead of interpreting queries, the system **compiles** them into optimized JavaScript functions, handling thousands of relational queries per second without CPU spikes.
3.  **Enterprise Multi-Tenancy (Isolation)**: We use **Contextual Isolation**. The `tenantId` is baked into the database adapter calls at the lowest level. It is mathematically impossible for Tenant A to query Tenant B's data because the ID is injected before the query is constructed.
4.  **Real-Time SSE (The "Efficiency" Tier)**: SveltyCMS uses Server-Sent Events (SSE) for AI token streaming and live collaboration. It works through Cloudflare and Load Balancers natively with a minimal RAM footprint compared to WebSockets.

## 📡 The Response Strategy

All REST endpoints follow a unified response shape to ensure consistent client-side handling.

### Success Format (200 OK / 201 Created)

```json
{
  "success": true,
  "data": { ... },
  "performance": { "duration": 4.5 }
}
```

### Error Format (4xx / 5xx)

```json
{
  "success": false,
  "error": "UNAUTHORIZED",
  "message": "Invalid Bearer token provided."
}
```

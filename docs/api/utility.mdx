# Utilities Reference (`utility.ts`)

The Utilities API provides a set of essential maintenance and support services that keep the SveltyCMS environment healthy and performant. It manages ephemeral data like caches, recoverable data like the Trash, and transactional communications like Email.

---

## ⚡ Quick Reference

| Feature           | HTTP Endpoint        | Method | Permission Required |
| :---------------- | :------------------- | :----- | :------------------ |
| **System Cache**  | `/api/cache/clear`   | `POST` | `manage:system`     |
| **Trash List**    | `/api/trash`         | `GET`  | `collection:delete` |
| **Restore Entry** | `/api/trash/restore` | `POST` | `collection:delete` |
| **Send Email**    | `/api/send-mail`     | `POST` | `manage:system`     |
| **Marketplace**   | `/api/marketplace`   | `GET`  | **Public**          |
| **OpenAPI Spec**  | `/api/openapi.json`  | `GET`  | **Public**          |

---

## 1. Maintenance & Cache

### System Cache Management

Manually invalidate the system-wide cache or clear specific categories (e.g., `collections`, `schemas`).

**Endpoint**: `POST /api/cache/clear`  
**Payload**: `{ "category": "all" }`

---

## 2. Recovery (The Trash)

SveltyCMS implements a "Soft Delete" pattern. Deleted entries are moved to the Trash before being permanently purged.

- **List Trash**: `GET /api/trash` — Returns a paginated list of items currently in the trash for the current tenant.
- **Restore**: `POST /api/trash/restore` — Returns an item to its original collection and restores its original status.
- **Empty Trash**: `DELETE /api/trash` — Permanently deletes all items in the trash.

---

## 3. Communication

### Transactional Email

Send Svelte-rendered emails directly via the API.

**Endpoint**: `POST /api/send-mail`  
**Payload**:

````json
{
  "recipientEmail": "user@example.com",
  "subject": "System Alert",
  "template": "alert",
  "props": { "message": "High CPU usage" }
}
```text
````

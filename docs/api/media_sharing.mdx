# 🌐 Media Sharing Endpoint API Reference

This endpoint is responsible for serving media assets that have been explicitly shared via a unique, time-limited token. It acts as the secure gateway to private content.

**Endpoint:** `GET /api/media/share`

**Purpose**: Validates access permissions (token existence, expiry, password) and delivers the file or an appropriate error status.

## 🔑 Parameters (Query Only)

| Parameter  | Type   | Required    | Description                                                                     | Notes                                                  |
| :--------- | :----- | :---------- | :------------------------------------------------------------------------------ | :----------------------------------------------------- |
| `id`       | String | Yes         | The unique internal ID of the media asset within SveltyCMS.                     | Must correspond to an existing asset record.           |
| `token`    | String | Yes         | The cryptographically secure, time-limited sharing token provided by the owner. | Required for all external shares.                      |
| `password` | String | Conditional | The password required if the shared link has password protection enabled.       | Only required if metadata indicates hashing is active. |

## ⚙️ Request Flow & Security Model

The server processes a multi-stage validation pipeline before streaming any content:

1.  **Existence Check**: Verify both `id` and `token`.
2.  **Token Validation**: Look up the shared link record associated with `token`. If not found or expired, access is denied immediately (`404` or `410`).
3.  **Password Gate (If Active)**: If `passwordRequired` flag is set on the link metadata, the provided `password` must be hashed and compared to the stored hash. Failure results in a specific error status code (`401`).

## 📤 Success Responses

### Status Code: `200 OK`

The request was successful, and the asset data is available. The response body typically includes metadata and initiates the download stream.

**Response Body Schema (Metadata):**

```json
{
  "success": true,
  "data": {
    "filename": "example-asset.pdf",
    "mimeType": "application/pdf",
    "size": 102400,
    "url": "/api/media/share?id=...&token=..." // The direct download URL for the client to use
  }
}
```

### Status Code: `200 OK` (Direct Download)

If no password is required, and the link is valid, the server streams the binary data directly without a JSON wrapper.

## 🛑 Error Responses

| HTTP Status        | Code / Message     | Description                                                                 | Action to Take                                   |
| :----------------- | :----------------- | :-------------------------------------------------------------------------- | :----------------------------------------------- |
| `400 Bad Request`  | Invalid Parameters | Missing required parameters (`id`, `token`) or incorrect format.            | User must check the link structure.              |
| `401 Unauthorized` | Incorrect Password | The provided password did not match the stored hash.                        | User must re-enter the correct credentials.      |
| `404 Not Found`    | Asset/Link Missing | No asset found matching `id`, or no share link found for the given `token`. | The asset ID or token is incorrect/non-existent. |
| `410 Gone`         | Link Expired       | The shared link was valid but its designated expiry date has passed.        | Content owner must re-issue the link.            |

## 📐 Technical Notes

- **Server Implementation**: Handled by the `load` function in `+page.server.ts`.
- **Security Impact**: This endpoint is highly sensitive and must always enforce strict tenant isolation checks on both the asset ID and token lookup to prevent cross-tenant data leakage.

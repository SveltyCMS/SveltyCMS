# Editable Website & Live Preview Plugin

Premium **Editable Website & Live Preview** plugin — bidirectional iframe sync between the CMS entry editor and your frontend. Works with the recommended [SvelteKit Site Starter + Svedit](/docs/development/site-starter.mdx) (same repo) or any decoupled frontend (Astro, Next.js, Vue) that implements the `postMessage` protocol.

| Live preview       | Real-time content preview in an iframe as you edit                                    |

| 14-day trial       | Full access for 14 days from first admin registration                                 |

## How It Works

1. **Collection schema** sets `livePreview` (URL pattern) and `plugins: ["editable-website"]`
2. **Preview URL** — `POST /api/preview/authorize` returns a signed `previewUrl`
3. **Handshake** — iframe sends `svelty:init`; CMS responds with `svelty:update`
4. **CMS → preview** — every form edit pushes fresh entry data to the iframe
5. **Preview → CMS** — field clicks focus the form; save messages merge into the entry (save in CMS to persist)

## Configuration

Set `livePreview` on your collection schema (string pattern or `true` for same-origin site starter):

```typescript
{
  name: "pages",
  livePreview: "/{slug}?lang={lang}",
  plugins: ["editable-website"],
  fields: [ /* ... */ ]
}
```

For external frontends (Astro, Next.js), use a full URL pattern:

```typescript
{
  livePreview: "https://myfrontend.com/{slug}?lang={lang}",
  fields: [ /* ... */ ]
}
```

### Site starter (recommended, free core + paid bridge)

The **Website Starter** preset seeds a `pages` collection, publishes a Svedit homepage, and **auto-enables** this plugin (trial starts on setup). The **public site is free**; the plugin unlocks **visual editing from the CMS**.

- `SITE_STARTER_ENABLED: true` (default) — guests see `/` with Svedit blocks
- `SITE_STARTER_ENABLED: false` — pure headless; `/` redirects to login
- **Without license/trial** — edit pages via the CMS form; Live Preview tab shows an upgrade prompt

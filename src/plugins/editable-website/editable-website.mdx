# Editable Website & Live Preview Plugin

Real-time bidirectional live preview with visual editing capabilities. Uses dynamic origin resolution from collection schema's `previewTargetUrl`, ephemeral preview token authorization, and a `postMessage` handshake protocol.

---

## Licensing & Pricing

**Price: €14.99** — **Fully paid plugin.** No free tier. A valid marketplace license key is required for all functionality.

A **14-day trial** is available from the first admin registration timestamp — no key required during trial. After the trial expires, the Live Preview tab is blocked and returns 403 until a valid license key is configured.

### License Verification

```typescript
import { checkExtensionLicense } from "@src/utils/license-manager";

const status = await checkExtensionLicense("plugin", "editable-website");
if (!status.active) {
  throw new Error("403 Forbidden: Active license required for Editable Website & Live Preview.");
}
```

## Features

| Feature             | Description                                                          |
| :------------------ | :------------------------------------------------------------------- |
| Live preview        | Real-time content preview in an iframe as you edit                   |
| Visual editing      | Click-to-edit elements directly in the preview                       |
| Dynamic origin      | Uses collection schema's `previewTargetUrl` — no hardcoded localhost |
| Preview tokens      | Ephemeral signed tokens for secure draft content access              |
| Handshake protocol  | `postMessage`-based communication between CMS and frontend           |
| Headless compatible | Works with Next.js, Astro, and any decoupled frontend                |
| Multi-collection    | Enable per-collection via `livePreview: true` schema flag            |
| 14-day trial        | Full access for 14 days from first admin registration                |

---

## How It Works

1. **Collection Schema** defines `previewTargetUrl` — the frontend URL to load in the iframe
2. **Preview Token** is generated via `/api/preview/authorize` for temporary draft access
3. **Handshake** — CMS sends a `postMessage` to the iframe; frontend responds to confirm readiness
4. **Bidirectional Sync** — Content changes in the editor push to the preview; clicks in the preview highlight fields in the editor

---

## Configuration

Set `livePreview: true` in your collection schema and provide a `previewTargetUrl`:

```typescript
{
  name: "blog_posts",
  livePreview: true,
  previewTargetUrl: "https://myfrontend.com/blog/{slug}?preview=true",
  fields: [ /* ... */ ]
}
```

# Slug Widget

The `Slug` widget is designed for creating URL-safe identifiers (slugs). It automatically normalizes input to lowercase and restricts characters to URL-safe patterns.

## Features

- **Automatic Normalization**: Converts input to lowercase and removes non-URL-safe characters.
- **Uniqueness Control**: Supports global or tenant-scoped uniqueness.
- **Multi-Tenancy**: Allows the same slug to exist across different tenants when scoped.
- **Validation**: Enforces URL-safe regex patterns (`a-z0-9-_`).

## Properties

| Property             | Type      | Default     | Description                                    |
| :------------------- | :-------- | :---------- | :--------------------------------------------- |
| `unique`             | `boolean` | `true`      | Whether the field must be unique.              |
| `disableUnique`      | `boolean` | `false`     | Force disable uniqueness check.                |
| `tenantScopedUnique` | `boolean` | `true`      | Scope uniqueness to the current tenant.        |
| `targetField`        | `string`  | `undefined` | The field name to auto-generate the slug from. |
| `translated`         | `boolean` | `false`     | Whether the slug is translatable.              |

## Multi-Tenancy Support

By default, the `Slug` widget sets `tenantScopedUnique: true`. This ensures that:

1. Two different tenants CAN have the same slug (e.g., both have a `/about` page).
2. Within a single tenant, slugs MUST remain unique.

To allow duplicate slugs even within a single tenant (not recommended), set `disableUnique: true`.

## Usage Example

```typescript
{
  widget: "Slug",
  label: "URL Path",
  db_fieldName: "slug",
  required: true,
  tenantScopedUnique: true,
  helper: "Used for the entry's URL"
}
```

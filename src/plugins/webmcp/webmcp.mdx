# WebMCP Server Gateway Plugin

Exposes controlled content management and navigation capabilities to remote AI agents via the Model Context Protocol (MCP). Functions as a headless server gateway with configurable transport modes (SSE/stdio), API key authentication, and token limit enforcement.

---

## 💰 Licensing & Pricing

**Price: €29.99** — **Fully paid plugin.** No free tier. A valid marketplace license key is required for all functionality.

A **14-day trial** is available from the first admin registration timestamp — no key required during trial. After the trial expires, the plugin stops functioning entirely until a valid license key is configured.

## Quick Start

1. **Purchase a license** at [marketplace.sveltycms.com](https://marketplace.sveltycms.com)
2. **Configure your license key** in System Settings
3. **Enable the plugin** in System Settings → Plugins
4. **Set transport mode** — choose `sse` for remote or `stdio` for local
5. **Generate an API key** for your AI agent
6. **Connect your agent** using the MCP endpoint

---

## Configuration

| Setting                | Type    | Required | Description                                           |
| :--------------------- | :------ | :------- | :---------------------------------------------------- |
| `transportMode`        | Public  | Yes      | Transport protocol: `sse` or `stdio` (default: `sse`) |
| `allowRemoteAgents`    | Public  | No       | Enable connections from external AI platforms         |
| `maxPayloadTokenLimit` | Public  | No       | Maximum token count per payload (default: `4096`)     |
| `agentApiKey`          | Private | Yes      | API key for authenticating AI agent connections       |

---

## Security

- **Draft-by-Default**: All content mutations from AI agents save as `draft` — human review required to publish
- **API Key Authentication**: Every agent connection requires a valid API key
- **Tenant Isolation**: Agents are scoped to their tenant context
- **Sandbox Compliance**: MCP tools operate within plugin sandbox boundaries (100 queries, 5s timeout)

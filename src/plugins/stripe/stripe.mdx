# Stripe Payments Plugin

Accept payments via Stripe for gated content, memberships, or e-commerce with PCI-compliant card collection. Uses the official Stripe SDK server-side and Stripe.js client-side for secure, isolated payment fields.

---

## Quick Start

1. **Enable the plugin** in System Settings → Plugins
2. **Configure API keys** in System Settings → Stripe (or set `STRIPE_SECRET_KEY` and `STRIPE_PUBLISHABLE_KEY` in settings)
3. **Add Stripe to a collection** by enabling it in the plugin's collection settings
4. **Collect payments** — Stripe Elements appear automatically on enabled collections

## Configuration

| Setting                  | Type    | Required | Description                                           |
| :----------------------- | :------ | :------- | :---------------------------------------------------- |
| `secretKey`              | Private | Yes      | Stripe secret key (`sk_live_...` or `sk_test_...`)    |
| `publishableKey`         | Public  | Yes      | Stripe publishable key (`pk_live_...`)                |
| `webhookSecret`          | Private | No       | Stripe webhook signing secret for verification        |
| `currency`               | Public  | No       | Default currency (default: `usd`)                     |
| `appearance`             | Public  | No       | Checkout theme (`default`, `stripe`, `night`, `flat`) |
| `allowedCheckoutOrigins` | Private | No       | CORS origins for headless frontend checkouts          |

---

## Security

- **PCI Compliance**: Card data is collected via Stripe Elements in an iframe — never touches your server
- **Webhook Verification**: All webhooks are verified using the signing secret
- **Audit Logging**: All payment transactions are logged to the audit trail
- **Draft-by-Default**: External webhook mutations save as draft pending review

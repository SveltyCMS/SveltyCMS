# Number Widget

The **Number Widget** is a specialized input component designed exclusively for numeric data entry. It stores values as actual number types, provides numeric range validation, and supports units via prefix/suffix fields.

## 🚀 Features

| Feature                  | Description                                                |
| ------------------------ | ---------------------------------------------------------- |
| ✅ **Native Storage**    | Stores data as actual `number` type for calculations       |
| ✅ **Range Validation**  | Schema-level `min` and `max` constraints                   |
| ✅ **Unit Support**      | Customizable `prefix` and `suffix` (e.g., "$", "kg")       |
| ✅ **Non-Translatable**  | Correctly treats numbers as universal, context-free data   |
| ✅ **Semantic HTML**     | Uses `type="number"` for optimized mobile keyboards        |
| ✅ **Locale Formatting** | Respects system language for thousands and decimal display |
| ✅ **Aggregations**      | Supports advanced filters (`$gt`, `$lt`, etc.) in queries  |
| ✅ **WCAG 2.2 AA**       | Full ARIA support, keyboard nav, and error associations    |

## 📦 Usage

### Basic Example

```typescript
import { widgets } from "@widgets";

export default {
  fields: [
    widgets.Number({
      label: "Stock Level",
      db_fieldName: "stock",
      required: true,
      min: 0,
      step: 1,
    }),
  ],
};
```

### With Units (Prefix/Suffix)

```typescript
widgets.Number({
  label: "Price",
  db_fieldName: "price",
  prefix: "$",
  suffix: "USD",
  min: 0,
  step: 0.01,
});
```

## ⚙️ Configuration

| Option     | Type      | Default | Description                      |
| ---------- | --------- | ------- | -------------------------------- |
| `label`    | `string`  | -       | Field display label (Required)   |
| `required` | `boolean` | `false` | Whether the field is mandatory   |
| `min`      | `number`  | -       | Minimum allowed numeric value    |
| `max`      | `number`  | -       | Maximum allowed numeric value    |
| `step`     | `number`  | `1`     | Numeric increment/decrement step |
| `prefix`   | `string`  | -       | Visual text before the input     |
| `suffix`   | `string`  | -       | Visual text after the input      |

## ♿ Accessibility

- **ARIA Labeled**: The input is explicitly associated with its label or an `aria-label`.
- **Error Description**: Validation errors are linked via `aria-describedby` for real-time screen reader feedback.
- **Constraint Feedback**: `min`, `max`, and `step` are natively handled and announced.
- **Interaction**: "Clear" button is keyboard reachable and labeled.
- **Focus Management**: Consistent focus rings and clear tab order.

## 🛠 Technical Implementation

### Validation Schema

Centralized in `index.ts` and refined in `input.svelte` using Valibot:

```typescript
const numberSchemaVal = $derived.by(() => {
  let schema: any = numberSchema("Value must be a number");
  if (typeof field.min === "number") schema = pipe(schema, minValue(field.min));
  if (typeof field.max === "number") schema = pipe(schema, maxValue(field.max));
  return field.required ? schema : nullable(schema);
});
```

### Aggregation Support

The widget translates numeric filters directly into database-optimized queries:

- `stock > 10` → `{ stock: { $gt: 10 } }`
- `price <= 100` → `{ price: { $lte: 100 } }`

### State Management

Uses **Svelte 5 Runes** for fine-grained reactivity:

- `$derived.by()`: Handles cross-language value resolution and schema generation.
- `$bindable()`: Ensures two-way synchronization with parent forms.
- `handleWidgetValidation`: Unified error handling bridge.

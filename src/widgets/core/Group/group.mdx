# Group Widget

<blockquote>
	**Structural widget** that allows you to organize multiple related fields into a single, logical block. Essential for creating clean, navigable
	content forms.
</blockquote>

## Overview

The **Group widget** acts as a container for other fields, allowing you to group related information together. Instead of a flat list of inputs, you can create sections like "SEO Settings", "Author Info", or "Product Details".

### Key Features

- ✅ **Field Organization**: Groups related fields visually and logically.
- ✅ **Collapsible Sections**: Reduce clutter by collapsing less frequently used groups.
- ✅ **Nested Data**: Saves data as a nested JSON object (`{ "group": { "field": "value" } }`).
- ✅ **Visual Variants**: Choose between `default`, `card`, or `bordered` styles.
- ✅ **Recursive Rendering**: Can contain any other widget, including other Groups.

## Usage Example

### SEO Settings Group

Group meta tags and search engine settings into a neat, collapsible section.

```typescript
import { widgets } from '@src/widgets/proxy';

export const PageSchema = {
	// ...
	fields: [
		widgets.Group({
			label: 'SEO Settings',
			db_fieldName: 'seo',
			icon: 'icon-park-outline:seo',
			collapsible: true,
			collapsed: true, // Start collapsed to save space
			variant: 'card',
			helper: 'Configure search engine optimization settings.',
			fields: [
				widgets.Input({
					label: 'Meta Title',
					db_fieldName: 'title',
					required: true,
					maxLength: 60
				}),
				widgets.Textarea({
					label: 'Meta Description',
					db_fieldName: 'description',
					maxLength: 160
				})
			]
		})
	]
};
```

## Configuration Properties

| Property       | Type              | Required | Default   | Description                                          |
| -------------- | ----------------- | -------- | --------- | ---------------------------------------------------- |
| `label`        | `string`          | ✅ Yes   | -         | Display label for the group header                   |
| `fields`       | `FieldInstance[]` | ✅ Yes   | `[]`      | Array of widget instances to render inside the group |
| `db_fieldName` | `string`          | ✅ Yes   | -         | Key for the nested object in the database            |
| `collapsible`  | `boolean`         | ❌ No    | `false`   | Allow users to collapse/expand the group             |
| `collapsed`    | `boolean`         | ❌ No    | `false`   | Initial state if collapsible                         |
| `variant`      | `string`          | ❌ No    | `default` | Visual style: `'default'`, `'card'`, or `'bordered'` |
| `icon`         | `string`          | ❌ No    | -         | Icon to display in the group header                  |
| `helper`       | `string`          | ❌ No    | -         | Helper text displayed below the header               |
| `width`        | `number`          | ❌ No    | `2`       | Grid width (1-2)                                     |
| `permissions`  | `object`          | ❌ No    | -         | Role-based permissions                               |

## Data Structure

The Group widget structures data hierarchically. It creates a nested object (JSON) within your content entry. This is supported natively in MongoDB (embedded documents) and mapped to JSON columns or relational tables in SQL adapters.

```json
// Stored Data
{
	"title": "My Landing Page",
	"seo": {
		"title": "Best Landing Page Ever",
		"description": "Join us today..."
	}
}
```

---

## Best Practices

1.  **Use for Logical Grouping**: Don't group unrelated fields just for aesthetics. Grouping implies a relationship.
2.  **Naming Conventions**: Use clear `db_fieldName` prefixes or nested structures to avoid collisions.
3.  **Depth Limit**: While recursive nesting is supported, avoid nesting more than 2-3 levels deep for usability.

# Tags Widget

<blockquote>
  **High-performance categorization tool** using a chip-based interface for managing lists of
  labels, categories, or keywords.
</blockquote>

## Overview

The **Tags widget** allows users to quickly add and remove labels from a document. It features a modern chip/badge UI, keyboard shortcuts (Enter/Comma to add, Backspace to remove), and robust duplicate prevention. It is the primary tool for building taxonomies within SveltyCMS.

### Key Features

- ✅ **Keyboard First**: Extremely fast entry using standard delimiter keys.
- ✅ **Duplicate Prevention**: Automatically detects and blocks duplicate tags (case-insensitive option).
- ✅ **Count Limits**: Enforce minimum and maximum tag requirements.
- ✅ **Interactive Chips**: Easily remove tags with one click.
- ✅ **A11y Compliant**: Full keyboard navigation and ARIA support.

---

## Basic Usage

### Article Keywords

```typescript
import Tags from "@src/widgets/custom/tags";

export const articlesCollection = {
  name: "Articles",
  fields: [
    Tags({
      label: "Keywords",
      db_fieldName: "keywords",
      allowDuplicates: false,
      caseSensitive: false,
      maxTags: 10,
      placeholder: "Add keyword...",
    }),
  ],
};
```

---

## Configuration Properties

| Property          | Type      | Required | Default | Description                            |
| ----------------- | --------- | -------- | ------- | -------------------------------------- |
| `allowDuplicates` | `boolean` | ❌ No    | `false` | Allow identical tags in the list       |
| `caseSensitive`   | `boolean` | ❌ No    | `false` | Treat "Apple" and "apple" as different |
| `maxTags`         | `number`  | ❌ No    | -       | Maximum number of tags allowed         |
| `placeholder`     | `string`  | ❌ No    | -       | Custom hint in the input field         |

## Data Structure

### Storage Format

Tags are stored as a clean array of strings.

```json
{
  "keywords": ["svelte", "cms", "ai", "high-performance"]
}
```

---

## Database Operations

### Filtering by Tag

MongoDB makes it easy to find documents containing specific tags:

```typescript
// Find any article containing the "svelte" tag
db.articles.find({ keywords: "svelte" });

// Find articles containing BOTH "svelte" and "cms"
db.articles.find({ keywords: { $all: ["svelte", "cms"] } });
```

# PhoneNumber Widget

The **PhoneNumber Widget** is a specialized input component designed for international telephone number entry. it enforces E.164 formatting by default and provides a clickable `tel:` link in display mode.

## 🚀 Features

| Feature                  | Description                                             |
| ------------------------ | ------------------------------------------------------- |
| ✅ **E.164 Validation**  | Hardened international phone format validation          |
| ✅ **Clickable Display** | Renders as a semantic `tel:` link with icon integration |
| ✅ **Semantic HTML**     | Uses `type="tel"` for optimized mobile keyboards        |
| ✅ **Custom Patterns**   | Supports project-specific regex overrides               |
| ✅ **Non-Translatable**  | Treats phone numbers as universal data                  |
| ✅ **WCAG 2.2 AA**       | Full ARIA support and error message association         |

## 📦 Usage

### Basic Example

```typescript
import { widgets } from "@widgets";

export default {
  fields: [
    widgets.PhoneNumber({
      label: "Support Line",
      db_fieldName: "phone",
      required: true,
    }),
  ],
};
```

### Custom Pattern

```typescript
widgets.PhoneNumber({
  label: "Local Office",
  db_fieldName: "office_phone",
  pattern: "^\\+49\\d{10}$", // Enforce German format
  placeholder: "+491234567890",
});
```

## ⚙️ Configuration

| Option        | Type      | Default       | Description                     |
| ------------- | --------- | ------------- | ------------------------------- |
| `label`       | `string`  | -             | Field display label (Required)  |
| `required`    | `boolean` | `false`       | Whether the field is mandatory  |
| `pattern`     | `string`  | E.164 Regex   | Custom regex validation pattern |
| `placeholder` | `string`  | `+1234567890` | Input placeholder text          |
| `icon`        | `string`  | -             | Iconify icon name               |

## ♿ Accessibility

- **Semantic Keyboard**: Uses `type="tel"` to trigger numeric keypads on mobile.
- **ARIA Labeled**: The input is explicitly associated with its label.
- **Error Association**: Validation errors are linked via `aria-describedby` for screen reader feedback.
- **Actionable Links**: The display component uses standard `<a>` tags with appropriate `tel:` protocols.

## 🛠 Technical Implementation

### Validation Schema

Defined in `index.ts` and utilized in `input.svelte` with Valibot:

```typescript
const phoneSchema = $derived.by(() => {
  const defaultPattern = /^\+?[1-9]\d{1,14}$/;
  const pattern = field.pattern ? new RegExp(field.pattern as string) : defaultPattern;
  const message = "Invalid phone number format (e.g., +1234567890)";
  const base = pipe(string(), regex(pattern, message));
  return field.required
    ? pipe(string(), minLength(1, "Phone number is required"), base)
    : optional(base, "");
});
```

### State Management

Uses **Svelte 5 Runes** for fine-grained reactivity:

- `$derived.by()`: Handles dynamic pattern and schema generation.
- `$bindable()`: Two-way synchronization with form state.
- `handleWidgetValidation`: Unified validation bridge for consistent error reporting.

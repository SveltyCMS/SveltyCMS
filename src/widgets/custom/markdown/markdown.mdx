# Markdown Widget

<blockquote>
  **Premium content editor** featuring a dual-pane interface for distraction-free writing with
  instant HTML preview.
</blockquote>

## Overview

The **Markdown widget** provides a middle ground between plain text and full Rich-Text editors. It is ideal for developers and technical writers who prefer the precision of Markdown while requiring immediate visual feedback. It supports all standard GFM (GitHub Flavored Markdown) syntax.

### Key Features

- âś… **Split-Pane UI**: Write on the left, see the result on the right.
- âś… **Instant Preview**: Optimized rendering engine with minimal latency.
- âś… **Translatable**: Full support for multi-language content models.
- âś… **Clean Output**: Stores raw Markdown strings, ensuring zero HTML pollution in your DB.
- âś… **Tooling Integrated**: Built-in support for bold, italics, links, and image syntax.

## Basic Usage

### Blog Post Body

```typescript
import Markdown from "@src/widgets/custom/markdown";

export const postsCollection = {
  name: "Posts",
  fields: [
    Markdown({
      label: "Body Content",
      db_fieldName: "body",
      translated: true,
      required: true,
      helper: "Use Markdown for rich content",
    }),
  ],
};
```

## Data Structure

### Storage Format

The widget stores a raw string (or a language-keyed object if translated).

```json
{
  "body": {
    "en": "# Hello World\nThis is **markdown**.",
    "de": "# Hallo Welt\nDas ist **Markdown**."
  }
}
```

## Best Practices

### âś… Use Markdown When:

- Writing technical documentation or blog posts.
- You need clean, portable data without HTML bloat.
- The audience is comfortable with Markdown syntax.

### âťŚ Don't Use Markdown When:

- Users need complex table builders or visual formatting tools (use `Rich-Text`).

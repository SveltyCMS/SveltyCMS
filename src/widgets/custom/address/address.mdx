# Address Widget

<blockquote>
  **Address input widget** with dual-mode interactive mapping: a completely free, zero-config
  OpenStreetMap-based stack by default, or an enterprise-grade Google Maps integration.
</blockquote>

## Overview

The **Address widget** captures and stores structured address data (street, house number, postal code, city, country, and latitude/longitude coordinates). It features a visual map interface with a draggable locator marker and address search autocomplete.

The widget dynamically supports two mapping engines:

1. **Default Mode (Free & Open Source)**: Uses **MapLibre GL JS** for rendering, **OpenFreeMap** for high-resolution vector tiles, and **Photon (Komoot)** for address search autocomplete and reverse geocoding. Requires no API keys or accounts.
2. **Enterprise Mode (Google Maps)**: Activates automatically when `publicEnv.GOOGLE_MAPS_API_KEY` is configured. Uses the **Google Maps JavaScript API**, **Google Places Autocomplete**, and **Google Geocoding**.

---

### Key Features

- ✅ **Structured Data**: Stores complete address object with separate fields for querying
- ✅ **Dual-Mode Mapping**: Fallback to MapLibre GL JS & OpenFreeMap if Google API key is absent
- ✅ **Geocoding & Autocomplete**: Autocomplete via Photon API (default) or Google Places
- ✅ **Draggable Marker**: Drags to pinpoint coordinates; automatically reverse-geocodes to fill in form inputs
- ✅ **Country Selection**: Multilingual country dropdown with ISO codes and localizations
- ✅ **Coordinate Capture**: Automatic latitude/longitude tracking
- ✅ **Field Visibility**: Configurable hidden fields for flexible layouts
- ✅ **Validation**: Valibot schema validation ensures completeness of required address fields
- ✅ **Universal Accessibility**: Keyboard navigable search suggestions and fully ARIA-compliant inputs

---

## When to Use Address Widget

### ✅ Use Address When:

- **Business Locations**: Store locations, branch offices, warehouses
- **Customer Data**: Shipping addresses, billing addresses
- **Event Management**: Venue addresses, meeting locations
- **Real Estate**: Property listings with map visualization
- **Delivery Services**: Pickup and delivery address management
- **Directory Listings**: Business directories, location finders

### ❌ Don't Use Address When:

- **Simple Text**: Use Input widget for unstructured address text
- **Coordinates Only**: Use Number widgets for just raw lat/lng
- **Regions Only**: Use Select widget for country/state selection without full address

## Configuration & Setup

### 1. Default (Free) Mode

No configuration is required! The widget uses the following open-source resources out-of-the-box:

- **Tiles**: `https://tiles.openfreemap.org/styles/bright/style.json`
- **Geocoding**: `https://photon.komoot.io/api/`
- **Reverse Geocoding**: `https://photon.komoot.io/reverse`

### 2. Google Maps Mode

If you prefer to use Google Maps (enterprise geocoding, street view, google autocomplete), simply configure your API key:

1. Obtain a Google Maps API Key from the [Google Cloud Console](https://console.cloud.google.com/).
2. Enable **Maps JavaScript API**, **Places API**, and **Geocoding API**.
3. Add the key to your SveltyCMS public configuration:

```typescript
// .env or configuration files
publicEnv.GOOGLE_MAPS_API_KEY = "AIzaSy...";
```

When SveltyCMS detects this key at runtime, the Address widget switches its entire map, search, and reverse-geocoding systems to Google Maps.

## Basic Usage

### Simple Address Form

```typescript
import Address from "@src/widgets/custom/Address";

export const businessesCollection = {
  name: "Businesses",
  fields: [
    Address({
      label: "Business Address",
      db_fieldName: "address",
      required: true,
      defaultCountry: "US",
      mapCenter: { lat: 40.7128, lng: -74.006 }, // New York
      zoom: 12,
    }),
  ],
};
```

### Omit coordinates or specific fields from UI

```typescript
Address({
  label: "Delivery Address",
  db_fieldName: "deliveryAddress",
  hiddenFields: ["latitude", "longitude", "houseNumber"],
  // Users won't see coordinate fields or house number
});
```

Available fields to hide:

- `'latitude'`
- `'longitude'`
- `'houseNumber'`

## Configuration Properties

| Property         | Type         | Required  | Default                   | Description                               |
| ---------------- | ------------ | --------- | ------------------------- | ----------------------------------------- |
| `label`          | `string`     | ✅ Yes    | -                         | Display label for the field               |
| `db_fieldName`   | `string`     | ✅ Yes    | -                         | Database field name                       |
| `required`       | `boolean`    | ❌ No     | `false`                   | Whether address is mandatory              |
| `defaultCountry` | `string`     | ❌ No     | `'DE'`                    | Default country (ISO 3166-1 alpha-2 code) |
| `mapCenter`      | `{lat, lng}` | ❌ No     | `{lat: 51.34, lng: 6.57}` | Default map center coordinates            |
| `zoom`           | `number`     | ❌ No     | `12`                      | Default map zoom level (1-20)             |
| `hiddenFields`   | `string[]`   | ❌ No     | `[]`                      | Fields to hide from UI                    |
| `width`          | `number`     | ❌ No     | `undefined`               | Field width in collection builder         |
| `translated`     | `boolean`    | 🔒 Locked | `false`                   | Always false (addresses are universal)    |

## Data Structure

Address data is stored as a **structured object** with all components:

```json
{
  "_id": "business_123",
  "name": "Acme Corp",
  "address": {
    "street": "Main Street",
    "houseNumber": "123",
    "postalCode": "90210",
    "city": "Beverly Hills",
    "country": "US",
    "latitude": 34.0736,
    "longitude": -118.4004
  }
}
```

### TypeScript Interface

```typescript
interface AddressData {
  street: string; // Street name
  houseNumber: string; // Building/house number
  postalCode: string; // ZIP/postal code
  city: string; // City name
  country: string; // ISO 3166-1 alpha-2 code (e.g. "US", "DE")
  latitude: number; // Decimal degrees latitude (-90 to 90)
  longitude: number; // Decimal degrees longitude (-180 to 180)
}
```

## Geospatial Queries

By storing coordinates as numeric fields, SveltyCMS enables high-performance spatial querying (e.g., in MongoDB, PostgreSQL, and MySQL/MariaDB).

### MongoDB 2dsphere Geospatial Indexing

```typescript
// Create geospatial index
db.businesses.createIndex({
  "address.location": "2dsphere",
});

// Find businesses within 5km of a coordinate [longitude, latitude]
db.businesses.find({
  "address.location": {
    $near: {
      $geometry: {
        type: "Point",
        coordinates: [-118.2437, 34.0522], // [lng, lat] order!
      },
      $maxDistance: 5000, // meters
    },
  },
});
```

## Address Formatting Examples

Different countries render address lines in varying order. SveltyCMS recommends parsing the structured fields into localized display strings:

### German Format

```typescript
`${address.street} ${address.houseNumber}\n${address.postalCode} ${address.city}`;
// Hauptstraße 123
// 10115 Berlin
```

### US Format

```typescript
`${address.houseNumber} ${address.street}\n${address.city}, ${address.country} ${address.postalCode}`;
// 123 Main Street
// Beverly Hills, US 90210
```

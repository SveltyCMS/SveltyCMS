# Geolocation Widget

<blockquote>
  **Coordinate selector widget** that stores data in standardized GeoJSON Point format, compatible
  with MongoDB geospatial indexes and mapping services.
</blockquote>

## Overview

The **Geolocation widget** provides an intuitive way to capture geographic coordinates (latitude and longitude). It integrates with the native **Browser Geolocation API** for one-click "Get Current Location" functionality and ensures data is stored in a way that enables advanced spatial queries like `$near` and `$geoWithin`.

### Key Features

- ✅ **GeoJSON Standard**: Stores data as `{ type: "Point", coordinates: [lng, lat] }`.
- ✅ **Native Geolocation**: One-tap access to device GPS/Location services.
- ✅ **High Precision**: Supports up to 6 decimal places for sub-meter accuracy.
- ✅ **Spatial Query Ready**: Optimized for MongoDB 2dsphere indexes.
- ✅ **Validation**: Ensures valid coordinate ranges (-90 to 90 for Lat, -180 to 180 for Lng).

---

## Basic Usage

### Standard Store Location

```typescript
import Geolocation from "@src/widgets/custom/geolocation";

export const storesCollection = {
  name: "Stores",
  fields: [
    Geolocation({
      label: "Store Location",
      db_fieldName: "location",
      required: true,
      defaultLat: 40.7128,
      defaultLng: -74.006,
      zoom: 15,
    }),
  ],
};
```

---

## Configuration Properties

| Property     | Type      | Required | Default | Description                         |
| ------------ | --------- | -------- | ------- | ----------------------------------- |
| `defaultLat` | `number`  | ❌ No    | `0`     | Initial latitude for the picker     |
| `defaultLng` | `number`  | ❌ No    | `0`     | Initial longitude for the picker    |
| `zoom`       | `number`  | ❌ No    | `12`    | Zoom level for mapping integrations |
| `required`   | `boolean` | ❌ No    | `false` | Whether location data is mandatory  |

---

## Data Structure

### Storage Format (GeoJSON)

```json
{
  "location": {
    "type": "Point",
    "coordinates": [-74.006, 40.7128]
  }
}
```

> [!IMPORTANT]
> GeoJSON stores coordinates in **[Longitude, Latitude]** order. The widget UI correctly maps these to the standard human-readable format.

---

## Database Operations

### Finding entries near a location

```typescript
db.stores.find({
  location: {
    $near: {
      $geometry: { type: "Point", coordinates: [-74.006, 40.712] },
      $maxDistance: 5000, // 5km
    },
  },
});
```

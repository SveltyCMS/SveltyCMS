### Security & Hardening (A++ Grade)
- **TOTP Replay Protection:** Implemented `UsedTotpCodeRegistry` with a 90s TTL to prevent 2FA replay attacks within the validity window.
- **Timing-Oracle Defense:** Pre-computed `DUMMY_ARGON2_HASH` and added fire-and-forget `verify()` to the user-not-found path in `signInUser()` to prevent email enumeration via response latency.
- **User Enumeration Defense:** Standardized the forgot password responses so that both success and failure paths return the same generic message.
- **CSRF-Safe 2FA:** Refactored the 2FA verification panel to use a hidden form with SvelteKit's `use:enhance` pipeline, closing the raw `fetch()` CSRF loophole.
- **Log Sanitation:** Prevented full Google profile JSON from leaking into debug logs; only the `sub` identifier is logged.
### GitHub OAuth Integration
- Added `GITHUB_CLIENT_ID` and `GITHUB_CLIENT_SECRET` to `private-config-schema.ts`.
- Created `github-auth.ts` to handle token exchange and user/email fetching directly from the GitHub API.
- Replaced monolithic `shouldShowOAuth` with distinct `shouldShowGoogleOAuth` and `shouldShowGithubOAuth`, enabling independent toggling of identity providers.
- Refactored `oauth-login.svelte` and `sign-up.svelte` to dynamically render distinct buttons for Google and GitHub.
- Updated `oauth/+page.server.ts` callback load function to dispatch to the appropriate provider (Google vs GitHub).
### Performance Optimizations
- **Static Imports on Hot Path:** Moved `auditLogService` and `argon2` to static top-level imports to eliminate cold-start latency during login.
- **Fire-and-Forget Logging:** Ensured all `auditLogService.log(...)` calls use `.catch()` so database writes never block the authentication response.
- **DB Health Cache:** Implemented a module-level 30-second TTL cache (`_dbHealthCache`) using lightweight `getRoleCount()` probes to prevent hammering the database on every load.
